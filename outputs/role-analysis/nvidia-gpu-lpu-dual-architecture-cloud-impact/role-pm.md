# PM 관점 분석: NVIDIA GPU+LPU 이중 구조 도입 — ThakiCloud 전략적 대응

## 관련도: 9/10

**분석 일자**: 2026-03-17

**근거**: GPU 클라우드 제공사의 핵심 제품 포지셔닝 변화. 신규 LPU 서비스 출시, 고객 페르소나 재정의, 로드맵·MVP·OKR 전반에 직접 영향. AI 에이전트 워크로드 급증과 TTFT SLO 요구가 제품 차별화 포인트로 부상.

---

## 1. 관련도 점수: 9/10

이 변화는 ThakiCloud 제품 로드맵에 **매우 높은 영향**을 미친다.

| 평가 차원 | 점수 | 근거 |
|-----------|------|------|
| 신규 제품 출시 | 10 | LPU 추론 서비스는 완전히 새로운 제품 라인 |
| 고객 페르소나 변화 | 9 | AI 에이전트·실시간 추론 고객층 신규 확보 |
| 로드맵 재조정 | 9 | 기존 GPU 중심 로드맵에 LPU 트랙 병행 필요 |
| 시장 포지셔닝 | 9 | TTFT 10ms vs 50–100ms는 명확한 차별화 포인트 |
| 수익 모델 | 8 | 추론 비용 구조 변화, 워크로드별 가격 책정 재설계 |

**종합**: LPU 도입은 단순 기능 추가가 아니라 **제품 포트폴리오 확장**이다. 2027년 LPX 출시·2026 말 LPU 랙 공급 시작 시점을 고려하면, 지금부터 PM 주도로 요구사항·MVP·로드맵을 정립해야 한다.

## 2. 사용자 스토리: LPU 인프라를 원하는 고객은 누구인가

### 페르소나 1: AI 에이전트 SaaS 기업 PM (에이전트 워크로드)

**페르소나**: AI 에이전트·멀티턴 대화형 서비스를 운영하는 B2B SaaS 기업의 제품 책임자. 토큰 수가 기존 챗봇 대비 15배 증가해 추론 비용이 급증하고 있다.

**JTBD**: "대화형 AI 응답 지연을 줄여 사용자 이탈을 막고, 동시에 추론 비용을 통제하고 싶다."

**사용자 스토리**:
- *"TTFT 50ms 이하를 보장하는 추론 엔드포인트를 선택할 수 있어야 한다."*
- *"에이전트 워크로드(짧은 시퀀스·고빈도 호출)에 맞는 가격 모델을 원한다."*

---

### 페르소나 2: MLOps 엔지니어 (실시간 추론 파이프라인)

**페르소나**: 금융·헬스케어 등 실시간 의사결정이 필요한 도메인의 MLOps 담당자. GPU 추론의 TTFT 50–100ms가 SLO를 위반한다.

**JTBD**: "실시간 추론 SLO(TTFT <20ms)를 충족하면서, 학습·파인튜닝은 기존 GPU 파이프라인으로 유지하고 싶다."

**사용자 스토리**:
- *"워크로드 유형(학습 vs 추론)에 따라 GPU/LPU를 자동 또는 수동으로 선택할 수 있어야 한다."*
- *"LPU 노드의 가용성·대기 시간을 모니터링하고 알림을 받을 수 있어야 한다."*

---

### 페르소나 3: 스타트업 CTO (비용 최적화)

**페르소나**: 시드/시리즈 A AI 스타트업 CTO. GPU 리스 비용이 부담되며, 추론 전용 워크로드에 더 저렴한 옵션을 찾고 있다.

**JTBD**: "추론 비용을 줄이면서 TTFT 성능은 유지하고 싶다."

**사용자 스토리**:
- *"GPU 대비 LPU 추론 단가를 비교하고, 워크로드별 비용 시뮬레이션을 할 수 있어야 한다."*
- *"소규모 트래픽부터 시작해 점진적으로 스케일할 수 있어야 한다."*

---

### 페르소나 4: 엔터프라이즈 인프라 담당자 (하이브리드 클라우드)

**페르소나**: 대기업 AI 플랫폼 담당자. 온프레미스 GPU와 클라우드 LPU를 조합한 하이브리드 아키텍처를 검토 중이다.

**JTBD**: "피크 추론 부하는 LPU로 오프로드하고, 민감 워크로드는 온프레미스 GPU로 유지하고 싶다."

**사용자 스토리**:
- *"LPU 클러스터와 기존 GPU 클러스터를 단일 콘솔에서 관리할 수 있어야 한다."*
- *"워크로드 라우팅 정책(지연·비용·규정 준수)을 설정할 수 있어야 한다."*

## 4. LPU 서비스 MVP 정의

### 최소 기능 범위 (MVP)

**목표**: 2026 Q4까지 LPU 추론 전용 엔드포인트를 1–2개 파일럿 고객에게 제공.

| 기능 | 포함 여부 | 수용 기준 |
|------|-----------|-----------|
| **LPU 전용 추론 API** | ✅ | REST/gRPC 엔드포인트, 모델 지정·배치 크기 설정 |
| **TTFT·토큰/초 메트릭** | ✅ | p50/p95 TTFT, 초당 토큰 처리량 노출 |
| **GPU vs LPU 선택** | ✅ | API 파라미터 또는 엔드포인트 경로로 선택 |
| **기본 모니터링 대시보드** | ✅ | LPU 노드 상태, 큐 대기 시간, 에러율 |
| **과금/사용량 추적** | ✅ | LPU 사용 시간·토큰 수 기반 미터링 |
| **자동 워크로드 라우팅** | ❌ (Later) | MVP에서는 명시적 선택만 지원 |
| **하이브리드 GPU+LPU 단일 배치** | ❌ (Later) | MVP에서는 LPU 전용 또는 GPU 전용만 |
| **멀티테넌시 고급 격리** | ❌ (Later) | MVP에서는 기본 네임스페이스 격리 |

### MVP 비기능 요구사항

- **지원 모델**: Groq 공식 지원 모델 (Llama, Mixtral 등) 3–5종
- **가용성**: 99% 목표 (파일럿 단계)
- **문서**: API 레퍼런스, 빠른 시작 가이드, GPU vs LPU 비교표

### MVP 제외 (의도적)

- 학습·파인튜닝 (GPU 전용 유지)
- 커스텀 모델 업로드 (Later)
- 에어갭/온프레미스 LPU (Later)
