# Codex Image to Eagle

Archive images generated by Codex into Eagle and preserve the generation prompt as searchable metadata (annotation).

## Prerequisites

1. **Eagle** is installed and running
2. **Eagle MCP plugin** is enabled (default `http://127.0.0.1:41596`)
3. **Node.js ≥ 18** is installed (required for built-in `fetch` API)
4. Codex-generated images exist in `~/.codex/generated_images` (or a custom directory)

For stdio MCP mode, configure `~/.codex/config.toml`:

```toml
[mcp_servers.eagle]
command = "node"
args = ["/path/to/Eagle/Plugins/mcp-server/modules/mcp-proxy.js"]
env = { EAGLE_SERVER_URL = "http://127.0.0.1:41596" }
```

## Script Location

The bundled script is at:

```
.cursor/skills/standalone/codex-image-to-eagle/scripts/archive-codex-image-to-eagle.js
```

## Workflow

1. **Pre-flight check** — verify Eagle is reachable:
   ```bash
   node .cursor/skills/standalone/codex-image-to-eagle/scripts/archive-codex-image-to-eagle.js \
     --check-connection
   ```
   If this fails, ask the user to start Eagle and enable the MCP plugin before proceeding.

2. **Identify images** — determine which images to archive:
   - Latest single image: `--latest`
   - All images from the latest generation session: `--latest-session`
   - A specific file: `--path "/absolute/path/image.png"`

3. **Capture the prompt** — get the exact prompt used to generate the image(s).
   The `--prompt` flag is **required**. If omitted, the script exits with code 1 and prints a usage hint. For long prompts, use `--prompt-file` instead.

4. **Dry-run first** — always preview before writing:
   ```bash
   node .cursor/skills/standalone/codex-image-to-eagle/scripts/archive-codex-image-to-eagle.js \
     --latest --prompt "PROMPT_TEXT" --dry-run
   ```

5. **Execute the import** — run without `--dry-run` after confirmation:
   ```bash
   node .cursor/skills/standalone/codex-image-to-eagle/scripts/archive-codex-image-to-eagle.js \
     --latest --prompt "PROMPT_TEXT"
   ```

6. **Report results** — return the imported Eagle item ID(s) and the folder used.

## Common Options

```bash
SCRIPT=".cursor/skills/standalone/codex-image-to-eagle/scripts/archive-codex-image-to-eagle.js"

# Import the newest Codex-generated image
node $SCRIPT --latest --prompt "PROMPT_TEXT"

# Import all images from the latest generation session
node $SCRIPT --latest-session --prompt "PROMPT_TEXT"

# Import a specific file
node $SCRIPT --path "/path/image.png" --prompt "PROMPT_TEXT"

# Read a long prompt from a file
node $SCRIPT --latest --prompt-file "/path/prompt.txt"

# Use stdio MCP transport instead of HTTP
node $SCRIPT --transport stdio --latest --prompt "PROMPT_TEXT"

# Test Eagle MCP connectivity
node $SCRIPT --check-connection

# Choose a target Eagle folder by name or ID
node $SCRIPT --latest --folder-name "Codex Generated Images" --prompt "PROMPT_TEXT"
node $SCRIPT --latest --folder-id "FOLDER_ID" --prompt "PROMPT_TEXT"

# Disable date subfolders (flat import into root folder)
node $SCRIPT --latest --no-date-subfolders --prompt "PROMPT_TEXT"

# Override the image source directory
node $SCRIPT --latest --image-dir "~/.codex/generated_images" --prompt "PROMPT_TEXT"
```

## Default Behavior

- **Image source**: `CODEX_GENERATED_IMAGES_DIR` env var, or `~/.codex/generated_images`
- **Eagle connection**: local Eagle MCP on `http://127.0.0.1:41596` (prefers IPv4 loopback over `localhost`)
- **Stdio mode**: reads `~/.codex/config.toml`, prefers `[mcp_servers.eagle]`, auto-detects other Eagle-like entries
- **Default tags**: `Codex`, `AI-Generated`, `Prompt`
- **Default folder**: `Codex Generated Images` with date subfolders (`Codex Generated Images/YYYY-M-D`)

## Prompt Annotation Format

The script writes this structure into the Eagle item's `annotation` field:

```text
Prompt:
<the full generation prompt>

Archive:
- source: Codex
- original_path: /path/to/original/image.png
- archived_at: 2026-04-27T00:00:00.000Z
```

## `--latest` vs `--latest-session`

- `--latest`: recursively scans the image directory and selects the single most recently modified image
- `--latest-session`: scans session subdirectories, picks the most recently updated session, and imports all images in that directory

### Example 1: Archive the latest image with its prompt

## Error Handling

| Error | Action |
|-------|--------|
| Eagle not running or MCP unreachable | Ask user to launch Eagle and enable the MCP plugin |
| No images found in source directory | Verify `--image-dir` path or `CODEX_GENERATED_IMAGES_DIR` env var |
| Duplicate import warning | Warn user; avoid re-running without `--dry-run` |
| stdio MCP config not found | Guide user to add `[mcp_servers.eagle]` to `~/.codex/config.toml` |
| Permission denied on image files | Check file permissions on the source directory |
| Node.js not installed | Ask user to install Node.js |

## Exit Codes

| Code | Meaning |
|------|---------|
| `0` | Success — all images imported |
| `1` | Usage error (missing `--prompt`, invalid arguments, no images found) |
| `1` | Runtime error (Eagle unreachable, MCP failure, file permission denied) |

## Safety

- **Always dry-run first** when the prompt, folder, or image selection is uncertain.
- **Never delete or move** source files in the Codex generated images directory.
- If Eagle is not running or the MCP plugin is disabled, ask the user to start Eagle and enable the Eagle MCP plugin before proceeding.
- Avoid repeated actual imports as they create duplicate items in Eagle.
