# Vibe-Trading Quantitative Analysis

## Overview

Three quantitative tools accessible via the `user-vibe-trading` MCP server:

1. **Factor Analysis** -- IC/IR and layered portfolio backtest
2. **Option Pricing** -- Black-Scholes with full Greeks
3. **Pattern Recognition** -- Technical chart pattern detection

---

## 1. Factor Analysis

Compute Information Coefficient (IC), Information Ratio (IR), and
top/bottom quintile return spreads for any factor.

### Usage

```
CallMcpTool server=user-vibe-trading toolName=factor_analysis
arguments={
  "codes": ["000001.SZ", "000002.SZ", "600519.SH", "601318.SH"],
  "factor_name": "pe_ttm",
  "start_date": "2023-01-01",
  "end_date": "2024-12-31",
  "source": "auto",
  "top_n": 10,
  "bottom_n": 10
}
```

### Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `codes` | list[str] | yes | | Stock universe |
| `factor_name` | string | yes | | Factor column name |
| `start_date` | string | yes | | YYYY-MM-DD |
| `end_date` | string | yes | | YYYY-MM-DD |
| `source` | string | no | `"auto"` | Data source |
| `top_n` | int | no | `10` | Top-ranked stocks |
| `bottom_n` | int | no | `10` | Bottom-ranked stocks |

### Supported Factors

- **Valuation**: `pe_ttm`, `pb`, `ps_ttm`, `pcf`
- **Momentum**: `pct_chg`, `ret_5d`, `ret_20d`, `ret_60d`
- **Liquidity**: `turnover_rate`, `volume_ratio`
- **Size**: `total_mv`, `circ_mv`

### Interpreting Results

| Metric | Meaning | Good Signal |
|--------|---------|-------------|
| IC (mean) | Average rank correlation with forward returns | |IC| > 0.03 |
| IC (std) | IC stability | Lower = more stable |
| IR | IC / IC_std | > 0.5 |
| Top quintile return | Return of highest-ranked stocks | > Bottom quintile |
| Long-short spread | Top - Bottom return | Positive and consistent |

## 2. Option Pricing (Black-Scholes)

Calculate theoretical option prices and Greeks.

### Usage

```
CallMcpTool server=user-vibe-trading toolName=analyze_options
arguments={
  "spot": 185.0,
  "strike": 190.0,
  "expiry_days": 30,
  "risk_free_rate": 0.05,
  "volatility": 0.25,
  "option_type": "call"
}
```

### Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `spot` | float | yes | | Current price |
| `strike` | float | yes | | Strike price |
| `expiry_days` | int | yes | | Days to expiration |
| `risk_free_rate` | float | no | `0.03` | Annual risk-free rate |
| `volatility` | float | no | `0.25` | Annual volatility (sigma) |
| `option_type` | string | no | `"call"` | `"call"` or `"put"` |

### Output

| Greek | Description | Use Case |
|-------|-------------|----------|
| Price | Theoretical option value | Fair value comparison |
| Delta | Price sensitivity to underlying | Directional exposure |
| Gamma | Delta sensitivity to underlying | Hedging adjustment frequency |
| Theta | Time decay per day | Theta strategy planning |
| Vega | Sensitivity to volatility changes | Vol trading |

### Integration with trading-options-strategy-advisor

Use `analyze_options` for **pricing calculations**, and the project's
`trading-options-strategy-advisor` skill for **strategy selection and P/L
simulation** (covered calls, iron condors, etc.).

## 3. Pattern Recognition

Detect classic chart patterns in OHLCV data.

### Usage

```
CallMcpTool server=user-vibe-trading toolName=pattern_recognition
arguments={"run_dir": "/tmp/vt-run"}
```

The `run_dir` must contain OHLCV CSV artifacts (typically generated by
a prior `backtest` call in `artifacts/ohlcv_*.csv`).

### Detected Patterns

| Category | Patterns |
|----------|---------|
| Reversal | Head-and-Shoulders, Inverse H&S, Double Top, Double Bottom |
| Continuation | Ascending/Descending Triangle, Symmetrical Triangle |
| Bilateral | Wedges (Rising/Falling), Channels (Ascending/Descending) |

### Output Fields

| Field | Description |
|-------|-------------|
| `pattern` | Pattern name |
| `direction` | `bullish` or `bearish` |
| `confidence` | Detection confidence (0-1) |
| `start_date` / `end_date` | Pattern time span |
| `target_price` | Projected target based on pattern measurement |

### Workflow: Backtest -> Pattern Overlay

```
1. Run backtest -> generates OHLCV artifacts in run_dir
2. Run pattern_recognition on same run_dir
3. Cross-reference detected patterns with backtest entry/exit points
4. Identify patterns that align with profitable trades
```
