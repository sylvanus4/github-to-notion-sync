# Korean OA Response — 의견서 및 보정서 작성

## Role

한국 특허 심사 거절이유통지에 대한 의견서(argument brief)와 보정서(amendment
document)를 작성하는 전문가 수준의 대응자. 보정 범위의 엄격한 제한(최초
출원 명세서 범위 내), 진보성 논증, 뒷받침 요건 보완을 중점적으로 처리.

## Prerequisites

- 의견제출통지서 (OA document from KIPO)
- 현재 청구항 및 명세서
- 심사관 인용 선행기술
- Write tool for persisting output to `outputs/patent-kr-oa/{date}/`

## Workflow

### Step 0: 입력 검증 (Input Validation)

| 항목 | 규칙 |
|------|------|
| **(a) 의견제출통지서 전문** | 통지서 **전문** 필수. **미제공 시 작업 중단(STOP)** — 거절이유를 추정·가정하지 말 것. |
| **(b) 현재 청구항 + 명세서 전문** | 보정·의견 작성을 위해 **청구항 전문**과 **명세서 전문**(또는 분석에 필요한 완전한 범위) 필수. |
| **(c) 의견제출기한** | 통지서상 **의견제출기한** 확인. **기한이 1개월 이내**이면 모든 산출물 상단에 **긴급(URGENCY) 경고**를 표시한다. |

### Step 1: 거절이유 분석

의견제출통지서에서 추출:

| 항목 | 내용 |
|------|------|
| 통지 유형 | 최초 거절이유 / 최후 거절이유 |
| 의견제출기한 | 통지일로부터 2개월 (연장 가능) |
| 거절 근거 | 청구항별: 조항, 사유, 인용문헌 |
| 기재불비 지적 | 구체적 지적 사항 |

거절이유 매트릭스:

| 청구항 | 거절 유형 | 법적 근거 | 인용문헌 | 핵심 쟁점 |
|--------|----------|----------|---------|----------|
| 1 | 진보성 부정 | 제29조 제2항 | 문헌 A + B | [결합 용이성] |
| 3 | 뒷받침 위반 | 제42조 제4항 제1호 | — | [명세서 근거 부족] |
| 5 | 명확성 위반 | 제42조 제4항 제2호 | — | [불명확 용어] |

### Step 2: 보정 전략 수립

각 거절이유에 대한 대응 전략:

**진보성 부정 (제29조 제2항) 전략**:
- 차이점 강조: 인용문헌과의 구성 차이 명확화
- 결합 곤란성 주장: 선행기술 간 결합 동기 부재
- 현저한 효과 주장: 예측할 수 없는 효과 입증
- 보정: 종속항 특징을 독립항에 추가 (범위 내)
- 기술분야 상이: 인용문헌의 기술분야 차이 주장

**뒷받침 요건 위반 (제42조 제4항 제1호) 전략**:
- 명세서 근거 제시: 해당 단락 번호 명시
- 보정: 청구항 범위를 명세서 기재 범위로 한정
- 명세서 보정 (최초 범위 내): 구체적 기재 보충

**기재불비 (제42조 제3항) 전략**:
- 실시 가능성 논증
- 명세서 보정으로 구체적 실시예 추가 (최초 범위 내에서만)

### Step 3: 보정서 (Amendment) 작성

**⚠️ 핵심 규칙: 보정 범위 제한**

한국 특허법 제47조에 따른 보정 범위:
1. **최초 출원 명세서에 기재된 사항의 범위 내**에서만 보정 가능
2. 신규사항(new matter) 추가 절대 불가
3. 최후 거절이유통지 후 보정은 더욱 제한적:
   - 청구항 삭제
   - 청구범위 감축
   - 오기 정정
   - 불명확한 기재의 명확화

보정서 형식:

```
【서류명】 보정서

【보정 대상 서류명】 특허청구범위

【보정항】 제1항

【보정 전】
[원래 청구항 전문]

【보정 후】
[보정된 청구항 전문]

【보정 이유】
상기 보정은 최초 출원 명세서 [단락 번호]에 기재된 사항을 근거로 하며,
[구체적 보정 이유 기재].
```

각 보정에 대해:
- 보정 근거 단락 번호 명시 (필수)
- 보정이 최초 명세서 범위 내임을 확인
- 보정으로 인한 청구범위 변화 설명

<!-- M1: 보정서 형식 검증 게이트 -->
**형식 검증 게이트:** 각 보정항 블록이 【보정항】·【보정 전】·【보정 후】·【보정 이유】 4개 필드를 **빠짐없이** 포함하는지 확인한다. 하나라도 누락된 보정항은 수정 완료 후에만 `amendment-kr.md`에 포함한다.

### Step 4: 의견서 (Argument Brief) 작성

의견서 구조:

```
【서류명】 의견서

【사건의 표시】 출원번호 10-20XX-XXXXXXX

1. 거절이유의 요지
   [심사관의 거절이유를 정확하게 요약]

2. 보정 내용
   [보정서의 보정 내용 요약]

3. 거절이유에 대한 의견

   가. 제29조 제2항 (진보성) 거절에 대하여

   (1) 인용문헌과의 차이점
       본원 발명의 청구항 1은 [차별점 요소]를 포함하고 있으나,
       인용문헌 1(문헌번호)에는 해당 구성이 개시되어 있지 않습니다.

   (2) 결합의 곤란성
       인용문헌 1과 인용문헌 2는 [기술 분야/목적/구성]이 상이하여,
       통상의 기술자가 이를 결합할 동기가 존재하지 않습니다.

   (3) 현저한 효과
       본원 발명은 [구체적 효과]의 현저한 향상을 달성하며,
       이는 인용문헌의 결합으로부터 예측할 수 없는 것입니다.

   나. 제42조 제4항 (기재요건) 거절에 대하여
       [보정으로 해소됨을 설명 / 명세서 근거 제시]

4. 결론
   이상과 같이 본원 발명은 거절이유를 해소하였으므로,
   특허결정하여 주시기를 요청합니다.
```

### Step 5: 최후 거절이유 대응

최후 거절이유통지의 경우 추가 검토:

| 대응 방안 | 설명 | 적용 조건 |
|----------|------|----------|
| 제한적 보정 | 삭제/감축/오기정정/명확화만 가능 | 기본 대응 |
| 재심사 청구 | 보정 후 재심사 요청 | 보정으로 거절 극복 가능 시 |
| 거절결정 불복심판 | 특허심판원에 불복 | 심사관 판단에 이견 시 |
| 분할출원 | 일부 청구항을 분할 | 복수 발명 포함 시 |

### Step 6: 기한 관리

| 항목 | 기한 | 비고 |
|------|------|------|
| 의견제출기한 | 통지일 + 2개월 | 연장 가능 (1회 1개월) |
| 보정 가능 기한 | 의견제출기한 내 | 최후거절 시 제한적 |
| 재심사 청구 | 거절결정등본 + 30일 | 보정 필수 동반 |
| 불복심판 | 거절결정등본 + 30일 | 재심사와 택일 |

### Step 7: Persist Output

Write to `outputs/patent-kr-oa/{date}/`:
- `oa-analysis-kr.md` — 거절이유 분석 매트릭스
- `amendment-kr.md` — 보정서
- `argument-kr.md` — 의견서
- `clean-claims-kr.md` — 보정 후 청구항
- `response-summary.json` — 구조화 요약

<!-- M4: 기한 표시 완전성 검증 -->
**기한 표시 게이트:** 위 5개 산출물 **각각**의 첫 3줄 이내에 `의견제출기한: YYYY-MM-DD` 줄이 있는지 확인한다. 누락 시 추가한 뒤 저장한다.

## Anti-Patterns (금지 사항)

1. 보정 시 **최초 명세서에 명시적·자명하게 기재되지 않은 사항**을 추가하지 말 것 — 위반 시 **보정각하** 사유가 될 수 있다.
2. 모든 거절이유에 대해 **동일한 논증을 반복**하지 말 것 — **거절이유별**로 쟁점·근거를 맞춘다.
3. 보정서에 **보정 근거 명세서 단락 번호**를 누락하지 말 것 — **각 보정**에 단락 근거를 명시한다.

<!-- M2: 단락 근거 완전성 검증 -->
4. 보정서 최종 검증 시 **모든 보정항**의 【보정 이유】 필드에 원 명세서 **단락 번호**(`[0042]` 등)가 1개 이상 인용되었는지 확인한다. 단락 번호 없는 보정항은 **보완 후 포함**한다.
4. **최후 거절이유** 대응 시 보정 범위를 초과하지 말 것 — **삭제·감축·오기정정·명확화** 등 허용 범위 내에서만 보정한다.

<!-- M3: 최후거절 보정 범위 자동 검증 -->
5. 최후 거절이유 대응 보정서 작성 시 각 보정항을 자동 분류한다: 삭제(D)·감축(N)·오기정정(C)·명확화(CL). 4개 범주에 해당하지 않는 보정항은 **경고 표시 후 삭제**한다.
5. **의견제출기한 경고**를 누락하지 말 것 — 모든 산출물에 기한을 표시한다.

## Worked Example: 보정서 항목 (테스트 발명)

**테스트 발명:** LLM 에이전트 오케스트레이션 — 스킬 레지스트리, 의미 검색, DAG, 자원 인지 모델 선택.

```
【보정항】 제1항

【보정 전】 …후보 에이전트 스킬 세트를 검색하는 단계…

【보정 후】 …스킬 레지스트리에 저장된 에이전트 스킬의 메타데이터와 상기 작업 기술 간의 임베딩 유사도를 계산하고, 상기 임베딩 유사도가 소정의 임계값 이상인 에이전트 스킬을 후보 에이전트 스킬 세트로 선택하는 단계…

【보정 이유】 상기 보정은 최초 출원 명세서 [0038]–[0040]에 기재된 '임베딩 유사도 임계값 기반 필터링'을 근거로 하며, 인용문헌 1과의 차별점을 명확화하기 위함.
```

(실제 출원에서는 단락·문헌 번호를 해당 사건에 맞게 대체한다.)

### Step 8: Pre-Delivery Check (전달 전 점검)

최종 전달 전에 다음을 **모두** 확인한다.

1. **거절이유별 대응:** 통지서의 **각 거절이유**에 대해 `oa-analysis-kr.md` 또는 요약에 **대응 전략**이 있는가?
2. **보정 근거:** **모든 보정**에 **명세서 단락 근거**가 명시되었는가?
3. **청구항 일관성:** `clean-claims-kr.md`가 보정서·의견서와 **일치**하고 종속 관계에 **모순**이 없는가?
4. **기한:** 모든 산출물 **상단**에 **의견제출기한**이 **눈에 띄게** 표시되었는가? (1개월 이내면 긴급 표시)
5. **최후 거절:** 최후 거절이유인 경우 보정이 **허용 범위**(삭제·감축·오기·명확화)를 **준수**하는가?

## Constraints

- 모든 산출물은 한국어로 작성

<!-- M5: 한국어 전용 출력 검증 -->
**한국어 전용 게이트:** 산출물 전체를 스캔하여 영문 단어가 5연속 이상 나타나는 줄을 검출한다. 해당 줄이 법조문·고유명사·인용 코드가 아닌 경우 **한국어로 교체** 후 저장한다. `response-summary.json`의 모든 value도 한국어여야 한다.

- 보정은 반드시 최초 출원 명세서 범위 내에서만 — 이 제한은 절대적
- AI 생성 문서는 변리사 검토 필수
- 기한 도과 시 출원 취하 간주 가능 — 기한 경고 항상 포함
- 최후 거절 시 보정 범위 더욱 제한됨을 명시

## Gotchas

- 한국의 보정 범위는 미국보다 훨씬 엄격: 미국은 continuation으로 새로운
  청구항을 추가할 수 있지만, 한국은 최초 명세서 범위 밖의 보정 불가
- "최초 출원 명세서에 기재된 사항"의 해석: 명시적 기재 + 자명한 사항까지
  포함하나, 해석 범위가 미국보다 좁음
- 최후 거절이유통지 후의 보정은 4가지로 제한 (삭제/감축/오기정정/명확화)
- 재심사 청구와 불복심판은 택일 관계 — 전략적 판단 필요
- 분할출원은 원출원이 계속 중인 동안에만 가능
- 진보성 논증에서 "후견적 고찰" (hindsight) 주장이 한국에서도 유효
- 기한 연장은 가능하나 횟수/기간 제한이 미국보다 엄격
