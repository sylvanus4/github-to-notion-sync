## Shell Aliases

### Context-switching aliases (changes active context)

```
kdev    → kubectx tkai-dev
kstage  → kubectx tkai-stage
kb200   → kubectx tkai-b200
kdemo   → kubectx tkai-demo
kmaster → kubectx tkai-master
kkata   → kubectx tkai-kata
```

### One-shot aliases (runs kubectl without switching)

```
kc-dev    → kubectl --context=tkai-dev
kc-stage  → kubectl --context=tkai-stage
kc-b200   → kubectl --context=tkai-b200
kc-demo   → kubectl --context=tkai-demo
kc-master → kubectl --context=tkai-master
kc-kata   → kubectl --context=tkai-kata
```

### Utility aliases

```
kx  → kubectx
kns → kubens
kls → kubecm list
```

## Adding a New Cluster

1. Obtain the kubeconfig file from the cluster admin
2. Copy to `~/.kube/<cluster-name>.config`
3. Run `kubecm add -f ~/.kube/<cluster-name>.config --context-name <name> --cover`
4. Rename if needed: `kubecm rename "<auto-generated>" "<desired>" -s`
5. Add aliases to `~/.zshrc`
6. Update this registry
