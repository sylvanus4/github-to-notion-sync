# 문서 품질 체크리스트

이 체크리스트를 교체하면 점검 기준이 변경된다 (Reviewer 패턴).

## PRD (Product Requirements Document)

### 구조 완성도 (8항목)
- [ ] 기능 개요 / 배경 섹션 존재
- [ ] 문제 정의 (Problem Statement) 명확
- [ ] 목표 사용자 / 페르소나 정의
- [ ] 유저 스토리 또는 유저 플로우 포함
- [ ] 기능 요구사항 (Functional Requirements) 명세
- [ ] 비기능 요구사항 (성능, 보안, 접근성) 명세
- [ ] 성공 지표 (KPI/OKR) 정의
- [ ] 일정 / 마일스톤 명시

### 상태값 커버리지 (5항목)
- [ ] State Matrix 또는 상태값 표 포함
- [ ] 모든 주요 화면의 상태 정의 (초기, 로딩, 성공, 에러, 빈 상태)
- [ ] 상태 전이 조건 명시
- [ ] 비로그인/권한별 상태 정의
- [ ] 비활성/비가용 상태 정의

### Edge Case (5항목)
- [ ] 에러 시나리오 정의
- [ ] 경계값 처리 명시
- [ ] 네트워크/성능 이슈 대응
- [ ] 동시성/충돌 시나리오
- [ ] 데이터 없음/대량 데이터 시나리오

### 정책 반영 (4항목)
- [ ] 관련 정책/규정 명시
- [ ] 정책별 반영 방식 설명
- [ ] 법적 필수 사항 체크 (개인정보, 약관 등)
- [ ] 정책 변경 시 영향 범위 명시

### 용어 일관성 (5항목)
- [ ] 동일 개념에 동일 용어 사용
- [ ] 약어 첫 등장 시 풀네임 표기
- [ ] 기술 용어에 비기술 설명 병기
- [ ] 팀 간 공통 용어 사전 참조
- [ ] TBD/TODO/placeholder 없음

## Design Spec (디자인 명세)

### 구조 완성도 (6항목)
- [ ] 컴포넌트 목록 및 계층 구조
- [ ] 디자인 토큰 참조 (색상, 타이포, 간격)
- [ ] 인터랙션 명세 (hover, click, transition)
- [ ] 반응형 브레이크포인트 정의
- [ ] 접근성 (a11y) 요구사항
- [ ] Figma 링크 포함

### 상태 커버리지 (4항목)
- [ ] 컴포넌트별 상태 정의 (default, hover, active, disabled, error)
- [ ] 데이터 상태별 UI (로딩, 빈 상태, 에러)
- [ ] 다크모드/라이트모드 대응
- [ ] 국제화 (i18n) 텍스트 길이 대응

## 일반 문서 (General)

### 기본 완성도 (5항목)
- [ ] 제목과 날짜 명시
- [ ] 목적/배경 설명
- [ ] 핵심 내용 구조화 (번호/불릿)
- [ ] 결론/다음 단계 명시
- [ ] 관련 링크/참조 문서

## 심각도 분류

| 심각도 | 기준 | 예시 |
|--------|------|------|
| Critical | 문서 목적 달성 불가 | 기능 요구사항 없는 PRD |
| High | 주요 누락으로 오해 발생 가능 | State Matrix 없음, Edge Case 전무 |
| Medium | 품질 저하지만 이해 가능 | 용어 불일치, 일부 TBD |
| Low | 개선 권장 | 포맷 불일치, 링크 깨짐 |
