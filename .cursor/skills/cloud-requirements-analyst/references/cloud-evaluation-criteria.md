# Cloud Evaluation Criteria

기획안의 기술적 타당성 평가 기준.

## 현실성 평가 차원

### 1. 기술 복잡도

| 점수 | 기준 |
|------|------|
| 5 | 관리형 서비스 조합으로 구현 가능 (Managed Service) |
| 4 | 약간의 커스텀 로직 필요, 표준 패턴 활용 |
| 3 | 상당한 커스텀 개발 필요, 사례는 존재 |
| 2 | 고도의 전문 기술 필요, 사례 드묾 |
| 1 | 연구 수준 기술, 상용 사례 거의 없음 |

### 2. 비용 효율성

| 점수 | 기준 |
|------|------|
| 5 | 프리 티어 또는 월 $100 이하로 운영 가능 |
| 4 | 월 $100–1,000 범위, 비즈니스 모델로 충분히 커버 |
| 3 | 월 $1,000–10,000, 스케일에 따라 조절 가능 |
| 2 | 월 $10,000+, 비용 최적화 필수 |
| 1 | 비용이 비현실적이거나 예측 불가 |

### 3. 운영 복잡도

| 점수 | 기준 |
|------|------|
| 5 | 서버리스/완전 관리형, 운영 부담 최소 |
| 4 | 기본 모니터링 + 알림 정도로 운영 가능 |
| 3 | 전담 DevOps 지원 필요 |
| 2 | 24/7 모니터링 및 전문 인력 필요 |
| 1 | 상시 전문 팀 필요, 장애 대응 복잡 |

### 4. 확장성

| 점수 | 기준 |
|------|------|
| 5 | Auto Scaling 기본 지원, 선형 확장 |
| 4 | 약간의 설정으로 확장 가능 |
| 3 | 아키텍처 변경 없이 10배 확장 가능 |
| 2 | 10배 확장 시 아키텍처 변경 필요 |
| 1 | 확장 한계가 명확하거나 재설계 필요 |

### 5. 구현 기간 (팀 3–5명 기준)

| 점수 | 기준 |
|------|------|
| 5 | 2주 이내 |
| 4 | 2–4주 |
| 3 | 1–2개월 |
| 2 | 3–6개월 |
| 1 | 6개월 이상 |

## 종합 현실성 점수 계산

```
종합 점수 = (기술복잡도 × 0.25) + (비용효율성 × 0.2) + (운영복잡도 × 0.15) + (확장성 × 0.2) + (구현기간 × 0.2)
```

| 종합 점수 | 판정 |
|-----------|------|
| 4.0–5.0 | 현실적: 표준 접근으로 구현 가능 |
| 3.0–3.9 | 조건부 가능: 특정 제약 하에 가능, 최적화 필요 |
| 2.0–2.9 | 도전적: 상당한 노력 필요, 대안 검토 권장 |
| 1.0–1.9 | 비현실적: 근본적 재검토 필요, 대안 필수 |

## 클라우드 서비스 매핑 가이드

### 일반 기능 → 서비스 매핑

| 기능 요구사항 | AWS | GCP | Azure |
|-------------|-----|-----|-------|
| 웹 애플리케이션 호스팅 | ECS/EKS, Lambda | Cloud Run, GKE | App Service, AKS |
| 관계형 데이터베이스 | RDS, Aurora | Cloud SQL | Azure SQL |
| NoSQL 데이터베이스 | DynamoDB | Firestore, Bigtable | Cosmos DB |
| 파일 저장 | S3 | Cloud Storage | Blob Storage |
| 실시간 메시징 | SNS/SQS, EventBridge | Pub/Sub | Service Bus |
| 검색 | OpenSearch | Vertex AI Search | Cognitive Search |
| 인증 | Cognito | Firebase Auth | Azure AD B2C |
| CDN | CloudFront | Cloud CDN | Azure CDN |
| ML/AI | SageMaker, Bedrock | Vertex AI | Azure AI |
| 모니터링 | CloudWatch | Cloud Monitoring | Azure Monitor |
