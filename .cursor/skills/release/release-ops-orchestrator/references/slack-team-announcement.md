## Thread 1 — 앱 오너가 할 일

```
:hammer_and_wrench: *앱 오너(개발자)가 해야 할 일*

*매주 월요일까지*:

1️⃣ *배포 대상 PR에 GitHub 라벨 4개 부착*
   • `release:approved` — 이번 주 목요일 배포 대상
   • `app:ai-platform` 또는 `app:agent-studio` — 앱 구분
   • `risk:low` / `risk:medium` / `risk:high` — 위험도
   • `qa:needed` — QA 필요 표시

2️⃣ *PR 본문에 릴리즈 템플릿 5개 섹션 작성*
   • Changes (변경 사항) — 어떤 걸 바꿨는지
   • User Impact (사용자 영향) — 사용자에게 어떤 영향이 있는지
   • QA Method (QA 방법) — 어떻게 테스트하면 되는지
   • Rollback Method (롤백 방법) — 문제 시 어떻게 되돌리는지
   • Related Issue/Ticket (관련 이슈) — 연결된 이슈 번호

3️⃣ *본인을 PR Assignee로 지정*

> :warning: *화요일 오전 10시까지 라벨이 없는 PR은 이번 주 배포에서 자동 제외됩니다.*
> :bulb: 한국어 또는 영어로 작성 가능합니다. 편하신 언어로 작성해주세요!
```

## Thread 2 — 7대 핵심 룰

```
📏 *7대 핵심 룰 — 꼭 기억해주세요!*

1️⃣ 화요일 오전까지 PR 없으면 → 이번 주 배포 제외
2️⃣ 수요일까지 QA 결과 없으면 → 목요일 배포 불가 (자동 Hold)
3️⃣ 핫픽스는 고객/비즈니스 영향 + 비즈니스팀 공유 여부 필수
4️⃣ 모든 배포 항목에 설명 가능한 앱 오너 필수
5️⃣ 배포 당일(목) 신규 항목 추가 금지 — 핫픽스만 예외
6️⃣ PR 템플릿 5개 섹션 모두 작성 필수
7️⃣ `release:approved`와 `hotfix` 라벨 동시 사용 금지

> :rotating_light: *핫픽스가 필요한 경우*: `#hotfix-alert` 채널을 사용해주세요.
> 정규 릴리즈와 핫픽스는 별도 트랙으로 관리됩니다.
```

## Thread 3 — 채널 안내 + 요일별 담당자

```

```

목  → 09AM 목록 잠금 → 14PM 배포 → 모니터링 → 개선점 3가지
