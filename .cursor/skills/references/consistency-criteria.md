## 1. 디자인 토큰 일관성

### 색상 (Color Tokens)

| 점검 항목 | 위반 기준 |
|-----------|----------|
| Semantic color 사용 | 제품 간 동일 의미에 다른 색상 토큰 → Critical |
| Brand color 적용 | 로고/강조색이 제품마다 다름 (의도적 차이 제외) → Major |
| 상태 색상 통일 | success/warning/error/info 색상이 다름 → Critical |
| 배경/표면 색상 | surface/background 토큰 불일치 → Major |

### 간격 (Spacing Tokens)

| 점검 항목 | 위반 기준 |
|-----------|----------|
| 기본 단위 | 4px/8px 배수 체계 통일 여부 → Critical |
| 섹션 간격 | 동일 레벨 섹션의 padding/margin 차이 → Major |
| 컴포넌트 내부 간격 | 동일 컴포넌트의 내부 padding 차이 → Major |

### 타이포그래피 (Typography Tokens)

| 점검 항목 | 위반 기준 |
|-----------|----------|
| 폰트 패밀리 | 제품 간 다른 폰트 사용 → Critical |
| 사이즈 스케일 | heading/body/caption 크기 체계 불일치 → Major |
| 줄 높이/자간 | 동일 토큰에 다른 line-height/letter-spacing → Minor |

## 2. 컴포넌트 사용 일관성

### 공통 패턴 매핑

제품 간 비교 대상이 되는 화면/패턴:

| 패턴 | 비교 항목 |
|------|----------|
| **로그인/인증** | 폼 구조, 에러 표시, 소셜 로그인 레이아웃 |
| **대시보드** | 카드 배치, 통계 표시, 차트 스타일 |
| **목록/테이블** | 정렬, 필터, 페이지네이션, 빈 상태 |
| **상세 페이지** | 탭 구조, 메타 정보 배치, 액션 버튼 위치 |
| **설정** | 폼 레이아웃, 토글/체크박스 사용, 저장 피드백 |
| **모달/다이얼로그** | 크기, 닫기 방법, 버튼 배치 (취소 좌/확인 우) |

### 컴포넌트 비교 체크리스트

- [ ] 동일 기능에 동일 TDS 컴포넌트 사용
- [ ] 컴포넌트 variant(size, color, state)가 동일 맥락에서 동일
- [ ] 아이콘 세트가 통일 (아이콘 라이브러리, 스타일, 크기)
- [ ] 버튼 계층이 동일 (Primary → Secondary → Tertiary 용례)

## 3. 인터랙션 패턴 일관성

| 패턴 | 통일 기준 |
|------|----------|
| **삭제 확인** | 모든 제품에서 동일한 확인 다이얼로그 패턴 사용 |
| **폼 저장** | 자동 저장 vs 수동 저장이 제품 간 통일 |
| **검색** | 검색 위치(헤더 vs 콘텐츠 영역), 실시간 vs 엔터 검색 통일 |
| **필터** | 필터 UI 패턴(드롭다운, 사이드바, 칩) 통일 |
| **정렬** | 테이블 헤더 클릭 정렬 동작 통일 |
| **페이지네이션** | 숫자형 vs 무한스크롤 vs Load More 통일 |
| **토스트/알림** | 위치(상단/하단), 지속 시간, 스타일 통일 |

## 4. 용어 및 레이블 일관성

| 점검 항목 | 위반 기준 |
|-----------|----------|
| 동일 개념 동일 용어 | "삭제" vs "제거", "생성" vs "추가" 혼용 → Major |
| 버튼 레이블 | 같은 동작에 다른 레이블 ("저장" vs "확인" vs "적용") → Major |
| 빈 상태 메시지 | 톤앤매너 불일치 (존댓말 혼용 등) → Minor |
| 에러 메시지 | 에러 메시지 구조/톤 차이 → Minor |
| 네비게이션 레이블 | 같은 수준의 메뉴 항목 네이밍 패턴 불일치 → Major |

## 5. 레이아웃 구조 일관성

| 점검 항목 | 위반 기준 |
|-----------|----------|
| GNB(전역 네비게이션) | 구조, 로고 위치, 메뉴 순서 차이 → Critical |
| LNB(로컬 네비게이션) | 사이드바 유무, 너비, 접기 동작 차이 → Major |
| 콘텐츠 영역 최대 너비 | max-width 설정 차이 → Minor |
| 헤더 영역 | 페이지 제목, 브레드크럼, 액션 버튼 배치 차이 → Major |
| 푸터 | 유무 및 구성 차이 → Minor |

---

## 6. 에러 및 상태 처리 일관성

| 점검 항목 | 위반 기준 |
|-----------|----------|
| 로딩 패턴 | 스켈레톤 vs 스피너 혼용 → Major |
| 빈 상태 | 일러스트 스타일, 메시지 구조 차이 → Major |
| 에러 페이지 (404, 500) | 디자인, 메시지, 복구 액션 차이 → Major |
| 네트워크 에러 | 오프라인/타임아웃 처리 패턴 차이 → Major |
| 권한 부족 | 접근 제한 화면 디자인 차이 → Minor |

## 일관성 점수 산정

각 카테고리별 위반 건수를 집계하여 일관성 점수를 산정:

| 등급 | 기준 | 판정 |
|------|------|------|
| A | Critical 0건, Major 2건 이하 | PASS — 높은 일관성 |
| B | Critical 0건, Major 3-5건 | PASS — 개선 필요 |
| C | Critical 1-2건 또는 Major 6건 이상 | FAIL — 즉시 대응 |
| D | Critical 3건 이상 | FAIL — 긴급 대응 |
