# Text Patterns — 문구 유형별 톤앤매너 가이드

## 기본 원칙
- 합니다체(존댓말) 기본
- 기술 용어는 풀어서 설명
- 행동 유도(CTA)를 에러/빈 상태 문구에 반드시 포함
- 변수는 `{변수명}` 형태로 표기

## 유형별 패턴

### 에러 — 네트워크/서버
- 톤: 공감 + 해결책
- 길이: 1-2문장
- 패턴: `[상황 설명]. [해결 방법]`
- 예: "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."

### 에러 — 입력값 검증
- 톤: 구체적 안내
- 길이: 1문장
- 패턴: `[필드명]을(를) [올바른 형식]으로 입력해 주세요.`
- 예: "이메일 주소를 올바른 형식으로 입력해 주세요."

### 에러 — 권한 부족
- 톤: 정중한 안내 + 대안
- 길이: 1-2문장
- 패턴: `[기능]에 대한 접근 권한이 없습니다. [대안 행동]`

### 에러 — 비즈니스 규칙 위반
- 톤: 이유 설명 + 대안
- 길이: 1-2문장
- 패턴: `[이유] 때문에 [행동]을(를) 할 수 없습니다. [대안]`

### 빈 상태 — 목록
- 톤: 행동 유도
- 길이: 1-2문장
- 패턴: `아직 [항목]이 없습니다. [CTA 행동]`

### 빈 상태 — 검색
- 톤: 대안 제안
- 길이: 1-2문장
- 패턴: `[검색어]에 대한 결과가 없습니다. [대안 행동]`

### 성공 — 액션 피드백
- 톤: 간결한 확인
- 길이: 1문장 이하
- 패턴: `[항목]이(가) [행동] 되었습니다.`

### 확인 — 파괴적 액션
- 톤: 결과 경고 + 되돌림 여부
- 길이: 1-2문장
- 패턴: `[항목]을(를) [행동]하시겠습니까? [되돌림 가능 여부]`

### 확인 — 일반
- 톤: 간결한 확인
- 길이: 1문장
- 패턴: `[행동]하시겠습니까?`

### 안내 — 기능 가이드
- 톤: 설명적
- 길이: 1-3문장
- 패턴: `[기능]을(를) 사용하면 [효과]. [사용 방법]`

### 안내 — 플레이스홀더
- 톤: 예시 형태
- 길이: 1줄
- 패턴: `예: [구체적 예시]`

### 안내 — 툴팁
- 톤: 한 줄 설명
- 길이: 1문장 이하
- 패턴: `[개념]은(는) [간단한 설명]입니다.`
