# Ambiguity Patterns

모호한 표현 탐지 패턴 사전. doc-quality-gate (Required sections / Clarity 신호)에서 사용.

## Category 1: 정량화 가능한데 정성적 표현

정량 기준을 제시할 수 있는데 모호하게 서술된 경우.

| 패턴 | 예시 | 개선안 |
|------|------|--------|
| "빠르게" / "신속히" | "빠르게 로딩한다" | "3초 이내 로딩한다" |
| "적절히" / "적절한" | "적절한 에러 메시지" | "HTTP 상태 코드별 에러 메시지 (표 참조)" |
| "충분히" / "충분한" | "충분한 여유 공간" | "최소 16px 간격" |
| "최대한" / "가급적" | "최대한 빨리 반영" | "요청 후 24시간 이내 반영" |
| "다수의" / "여러" | "여러 건의 알림" | "최대 10건의 알림" |
| "적은" / "많은" | "적은 수의 항목" | "5개 이하의 항목" |
| "간단히" / "쉽게" | "간단히 설정" | "3단계 설정: ① → ② → ③" |

## Category 2: 주어 부재 (Agentless Sentences)

행위의 주체가 불분명한 수동태/무주어 문장.

| 패턴 | 예시 | 개선안 |
|------|------|--------|
| "~된다" / "~처리된다" | "데이터가 처리된다" | "서버가 데이터를 처리한다" / "배치 스크립트가 처리한다" |
| "~시킨다" (주어 없음) | "알림을 발송시킨다" | "[Push 서버]가 알림을 발송한다" |
| "~해야 한다" (누가?) | "검증해야 한다" | "[프론트엔드]에서 입력값을 검증한다" |
| "~할 수 있다" | "조회할 수 있다" | "[관리자 권한 사용자]가 조회할 수 있다" |

## Category 3: 조건 없는 분기

분기 조건이 명시되지 않아 구현자가 임의로 판단해야 하는 경우.

| 패턴 | 예시 | 개선안 |
|------|------|--------|
| "경우에 따라" | "경우에 따라 노출하지 않음" | "사용자 등급이 Basic인 경우 노출하지 않음" |
| "상황에 맞게" | "상황에 맞게 처리" | "재고 부족 시 → 품절 표시, 배송 불가 시 → 주문 불가 표시" |
| "필요시" / "필요한 경우" | "필요시 재시도" | "API 응답 5xx 에러 시 최대 3회 재시도 (2초 간격)" |
| "등" / "기타" | "이름, 이메일 등" | "이름, 이메일, 전화번호 (전체 필드 목록은 표 참조)" |

## Category 4: 용어 불일치

동일 개념을 다른 용어로 지칭하여 혼란을 유발.

| 탐지 방법 | 설명 |
|----------|------|
| 동의어 클러스터링 | 문서 내 유사 용어를 그룹핑하여 불일치 탐지 |
| 정의 섹션 역참조 | 정의된 용어와 본문 사용 용어 대조 |
| 영한 혼용 탐지 | 같은 개념의 영어/한국어 혼용 (예: "사용자"와 "User" 병행) |

**일반적 불일치 패턴**:
- "사용자" vs "유저" vs "고객" vs "회원"
- "주문" vs "오더" vs "거래"
- "결제" vs "페이먼트" vs "과금"
- "알림" vs "노티피케이션" vs "푸시"
- "취소" vs "해지" vs "철회" vs "해약"

## Category 5: 팀 간 해석 차이 유발 표현

기획·디자인·개발 조직이 서로 다르게 읽을 수 있는 서술.

| 패턴 | 기획 해석 | 개발 해석 | 디자인 해석 |
|------|----------|----------|------------|
| "자연스럽게 전환" | 사용자가 불편 없이 | 페이지 리다이렉트 | 애니메이션 트랜지션 |
| "실시간으로 반영" | 즉시 사용 가능 | WebSocket/SSE 구현 | 로딩 없이 UI 갱신 |
| "사용자 친화적 에러" | 이해하기 쉬운 문구 | 에러 코드 매핑 | 에러 UI 컴포넌트 |
| "반응형 대응" | 모바일도 지원 | 미디어 쿼리 구현 | 브레이크포인트별 디자인 |

## 점수 계산 방법

```
모호 표현 비율 = 탐지된 모호 표현 수 / 전체 문장 수
명확성 점수 = (1 - min(모호 표현 비율, 1.0)) × 25
```

가중치: Category 5 (팀 간 차이) > Category 3 (조건 없는 분기) > Category 1 (정성적) > Category 2 (주어 부재) > Category 4 (용어 불일치)
