# 정책 준수 체크리스트

문구 유형별 필수 점검 항목. 이 체크리스트를 교체하면 점검 기준이 변경된다.

## 공통 점검 항목

- [ ] 금지 표현 미사용 (보장, 확실, 절대, 무조건 등 절대적 표현)
- [ ] 과장/허위 표현 미포함
- [ ] 공식 용어 사용 (사내 용어 사전 기준)
- [ ] 맞춤법/문법 정확성
- [ ] 대상 사용자에게 적절한 난이도

## UI 문구 (ui-copy)

- [ ] 동작 결과를 명확히 전달
- [ ] 16자 이내 (버튼), 40자 이내 (설명)
- [ ] 능동형 표현 사용
- [ ] 일관된 경어체/비경어체

## 에러 메시지 (error-message)

- [ ] 에러 원인 명시
- [ ] 해결 방법 안내 포함
- [ ] 사용자 탓으로 돌리지 않는 톤
- [ ] 기술 용어 미노출 (에러 코드 별도)
- [ ] 법적 고지 사항 포함 (해당 시)

## 약관 (terms)

- [ ] 법적 필수 고지 사항 포함
- [ ] 이용자 권리/의무 명시
- [ ] 면책 범위 명확
- [ ] 관할 법원/분쟁 해결 절차 명시
- [ ] 시행일/개정일 표기

## 안내문 (notice)

- [ ] 목적/배경 명시
- [ ] 영향 범위 안내
- [ ] 일정/기한 명시
- [ ] 문의처 안내
- [ ] 대체 방법 안내 (해당 시)

## 점검 결과 형식

각 항목에 대해 다음 형식으로 결과를 기록:

| 항목 | 결과 | 비고 |
|------|------|------|
| [체크 항목] | Pass / Fail | [Fail 시 사유와 수정 제안] |

**준수율**: Pass 항목 수 / 전체 항목 수 × 100%
