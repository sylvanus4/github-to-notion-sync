# Spec Templates

spec-assembler Phase 4에서 사용하는 문서 템플릿. 템플릿별 필수 섹션과 구조 정의.

## PRD Template

```markdown
# [제품/기능명] PRD

> **상태**: Draft | Review | Approved
> **작성자**: [이름]
> **최종 수정**: [날짜]
> **소스**: [출처 목록]

## 1. 개요

### 1.1 배경 및 목적
[왜 이 기능/제품이 필요한가]

### 1.2 문제 정의
[해결하려는 구체적 문제]

### 1.3 목표 및 KPI
| 목표 | KPI | 목표치 | 측정 방법 |
|------|-----|--------|----------|

## 2. 사용자 시나리오

### 2.1 타겟 사용자
[페르소나 또는 사용자 세그먼트]

### 2.2 주요 시나리오 (Happy Path)
1. ...
2. ...

### 2.3 대안 시나리오

### 2.4 엣지 케이스
| 상황 | 예상 동작 | 비고 |
|------|----------|------|

## 3. 요구사항

### 3.1 기능 요구사항
| 우선순위 | 요구사항 | 수용 기준 |
|----------|---------|----------|
| Must     |         |          |
| Should   |         |          |
| Could    |         |          |

### 3.2 비기능 요구사항
[성능, 보안, 접근성, 호환성]

### 3.3 정책/규칙 제약
[적용되는 정책서, 약관 조항]

## 4. 상세 설계

### 4.1 상태 정의
| 상태 | 설명 | 전이 조건 | 전이 후 상태 |
|------|------|----------|------------|

### 4.2 화면 흐름
[화면 전환 다이어그램 또는 설명]

### 4.3 데이터 요구사항
[필요 데이터 필드, 유효성 규칙]

### 4.4 API 요구사항 (해당 시)
[엔드포인트, 요청/응답 형식]

## 5. 일정 및 마일스톤
| 마일스톤 | 목표일 | 담당 |
|----------|--------|------|

## 6. 의존성 및 리스크
| 유형 | 내용 | 영향도 | 대응 |
|------|------|--------|------|

## 7. 미결 사항
| # | 항목 | 의사결정 필요 대상 | 기한 |
|---|------|-----------------|------|

## 부록: 소스 매핑
| 섹션 | 출처 | 비고 |
|------|------|------|
```

## Spec Template (기능 명세서)

```markdown
# [기능명] 기능 명세서

## 1. 기능 개요
[기능 설명, 적용 범위]

## 2. 사용자 흐름
[단계별 사용자 행동 + 시스템 응답]

## 3. 상태 정의
[상태값 테이블 + 전이 다이어그램]

## 4. 화면 정의
[화면별 구성 요소, 인터랙션]

## 5. 비즈니스 규칙
[조건별 처리 규칙 테이블]

## 6. 예외 처리
[에러 시나리오 + 사용자 안내]

## 7. 데이터 명세
[필드 정의, API 계약]

## 8. 테스트 시나리오
[주요 테스트 케이스]
```

## Policy Template (정책서)

```markdown
# [정책명]

## 1. 적용 범위
## 2. 용어 정의
## 3. 기본 원칙
## 4. 세부 규칙
### 4.1 [규칙 카테고리 1]
### 4.2 [규칙 카테고리 2]
## 5. 예외 사항
## 6. 시행일 및 변경 이력
## 7. 담당자 및 문의
```

## Brief Template (간략 기획서)

```markdown
# [제목] 기획 브리프

## 한 줄 요약
## 배경
## 핵심 요구사항 (3-5개)
## 제약 조건
## 예상 일정
## 다음 단계
```
