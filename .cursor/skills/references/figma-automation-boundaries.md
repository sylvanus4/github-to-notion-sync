# Figma 디자인-to-코드 자동화: 범위와 한계

## 자동화 가능한 영역

| 영역 | 설명 | 자동화 수준 |
|------|------|-----------|
| **디자인 토큰 추출** | 색상, 타이포그래피, 간격, 그림자 → CSS 변수 / 테마 파일 | 높음 |
| **컴포넌트 구조 변환** | Auto Layout 기반 레이어 → JSX/HTML 구조 (Flex/Grid) | 중간 |
| **반응형 브레이크포인트** | Constraints → Media Query | 중간 |
| **에셋 추출** | 아이콘, 이미지 → SVG/PNG export 및 코드 import | 높음 |
| **Variant → Props 매핑** | Boolean/enum variant → 컴포넌트 props | 중간 |

## 사람이 반드시 개입해야 하는 영역

| 영역 | 이유 |
|------|------|
| **비즈니스 로직** | 조건부 렌더링, 데이터 바인딩, API 연동은 디자인에 표현되지 않음 |
| **상태 관리** | 로딩/에러/빈값/비활성 상태 — Figma에서 완전히 정의하기 어려움 |
| **복잡한 인터랙션** | 드래그앤드롭, 제스처, 단계별 애니메이션은 Figma 프로토타입으로 불충분 |
| **접근성** | ARIA 역할, 키보드 네비게이션, 스크린 리더 호환은 디자인에 미표현 |
| **성능 최적화** | 가상화, 레이지 로딩, 번들 최적화는 구현 영역 |
| **Edge Case UI** | 텍스트 오버플로우, 동적 콘텐츠 사이징, 다국어 레이아웃 |

## 실제 운영 수준의 한계

### 네이밍 의존성
Figma 레이어명이 일관되지 않으면 코드 품질이 급격히 떨어집니다. `Frame 42`, `Group 17` 같은 자동 생성 이름은 의미 없는 코드를 생성합니다. 시맨틱 네이밍(예: `HeaderNav`, `PriceLabel`)이 전제 조건입니다.

### 1:1 매핑 불가
Figma 레이어 구조는 코드 컴포넌트 트리와 다릅니다. Figma에서는 시각적 그룹핑이지만, 코드에서는 로직/재사용성 기반 분리입니다. Wrapping, composition, render props 같은 코드 패턴은 디자인에서 추론 불가합니다.

### 상태 정의 부재
Figma는 주로 정상(happy path) 상태만 디자인합니다. 에러, 로딩, 빈값, 비활성 상태는 별도 프레임으로 정의해야 하지만 종종 누락됩니다.

### 토큰 파이프라인 필요
Figma → Code 단방향 파이프라인 구축이 필수입니다. 양방향 동기화는 충돌 해결 비용이 높아 비현실적입니다. Style Dictionary, Tokens Studio 같은 도구가 필요합니다.

### 정적 레이아웃 vs 동적 콘텐츠
고정 텍스트 기반 디자인은 잘 변환되지만, 동적 길이 텍스트, 가변 리스트, 사용자 생성 콘텐츠에서 레이아웃이 깨집니다.

### 프로토타입 vs 프로덕션
Figma 프로토타입의 화면 전환과 인터랙션은 실제 코드 로직(라우팅, 상태 관리, API 호출)과 크게 다릅니다.

## 권장 전략

```
Level 1 (즉시 가능):
  - 토큰 추출 + 에셋 export 자동화
  - Style Dictionary / Tokens Studio 파이프라인

Level 2 (조건부 가능):
  - Auto Layout 기반 컴포넌트 → 정적 코드 생성
  - 전제: 시맨틱 네이밍 규칙 준수, variant 체계 정립

Level 3 (인간 필수):
  - 상태 관리, 비즈니스 로직, 인터랙션
  - 코드 생성 후 반드시 수동 보완
  - 기획자/개발자 리뷰 필수
```

## 도구 생태계

| 도구 | 역할 | 자동화 수준 |
|------|------|-----------|
| Figma API / MCP | 디자인 데이터 추출 | 높음 |
| Style Dictionary | 토큰 → 코드 변환 | 높음 |
| Tokens Studio | Figma 토큰 관리 | 높음 |
| Storybook | 컴포넌트 문서화/테스트 | 중간 |
| Chromatic | 시각적 회귀 테스트 | 중간 |
| Figma Code Connect | 디자인-코드 연결 | 중간 |
