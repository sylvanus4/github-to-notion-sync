# Policy Categories

정책 유형별 분류 체계. 문구 생성 시 적용할 정책 조항을 빠르게 필터링하기 위한 참조.

## 정책 유형 분류

| 카테고리 | 하위 정책 | 문구 적용 영역 |
|----------|----------|---------------|
| **과금 정책** | 결제 수단, 요금제, 환불, 할인, 크레딧 | 결제 화면, 요금 안내, 영수증, 에러 |
| **이용약관** | 서비스 범위, 이용 제한, 면책, 분쟁 해결 | 가입 화면, 약관 동의, 법적 고지 |
| **개인정보** | 수집 항목, 이용 목적, 보유 기간, 제3자 제공 | 동의 화면, 설정, 탈퇴 안내 |
| **서비스 제한** | API 제한, 저장 용량, 사용량 쿼터, 기능 제한 | 대시보드, 알림, 업그레이드 안내 |
| **보안 정책** | 인증, 권한, 데이터 보호, 접근 제어 | 로그인, 2FA, 비밀번호 정책 안내 |
| **콘텐츠 정책** | 금지 콘텐츠, 저작권, 신고 절차 | 업로드 안내, 경고, 제재 알림 |
| **SLA** | 가용성 보장, 장애 대응, 보상 | 상태 페이지, 장애 공지 |

## 문구 유형 × 정책 매핑

| 문구 유형 | 주로 참조하는 정책 |
|-----------|------------------|
| UI 레이블 | 서비스 제한, 과금 정책 |
| 안내 메시지 | 서비스 제한, 이용약관, 보안 정책 |
| 에러 메시지 | 과금 정책, 보안 정책, 서비스 제한 |
| 고객 응대 | 이용약관, 과금 정책, SLA |
| 약관 문구 | 이용약관, 개인정보, 과금 정책 |

## 필수 포함 키워드 예시

각 정책 유형에서 반드시 사용해야 하는 법적/브랜드 용어:

- **과금**: "결제", "환불", "청구", "크레딧" (비격식: "돈 돌려받기" 금지)
- **개인정보**: "개인정보", "동의", "처리", "보유" (비격식: "정보 가져가기" 금지)
- **SLA**: "가용성", "보상", "장애" (비격식: "서버 죽었어요" 금지)
