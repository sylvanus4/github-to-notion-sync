# MiroFish Simulation Parameters

## Simulation Creation Parameters

```json
{
  "project_id": "string (REQUIRED) — Project ID from ontology generation",
  "graph_id": "string (optional) — Inferred from project if omitted",
  "enable_twitter": "boolean (default: true)",
  "enable_reddit": "boolean (default: true)"
}
```

> **Note:** `prediction_requirement` and `num_rounds` are auto-generated by the LLM during the `prepare` step. They are NOT passed to `create`.

## Agent Profile Structure

Each generated agent has:

| Field | Description |
|-------|-------------|
| `name` | Agent's name (extracted from knowledge graph entity) |
| `biography` | Detailed background story |
| `personality` | Big Five personality traits + behavioral tendencies |
| `social_relations` | Relationships with other agents (friend, rival, colleague, etc.) |
| `behavioral_logic` | Decision-making patterns and value systems |
| `memory` | Long-term memory stored in Zep Cloud |
| `platform_behavior` | How the agent behaves on simulated Twitter/Reddit |

## Simulation Scripts

MiroFish supports 3 simulation modes via backend scripts:

| Script | Path | Description |
|--------|------|-------------|
| `run_twitter_simulation.py` | `backend/scripts/` | Twitter-style dual-platform simulation |
| `run_reddit_simulation.py` | `backend/scripts/` | Reddit-style forum simulation |
| `run_parallel_simulation.py` | `backend/scripts/` | Parallel simulation across both platforms |

## Round Budget Guidelines (Measured 2026-03-16)

| Rounds | Duration | Actions | Token Cost | Use Case |
|--------|----------|---------|-----------|----------|
| 5-10 | ~15 min | ~50 | ~$0.5-1 | Quick proof-of-concept |
| **20** | **~35 min** | **~170** | **~$1-3** | **Daily stock analysis (recommended)** |
| 30-40 | ~60 min | ~350 | ~$3-8 | Deep analysis, complex scenarios |
| 50+ | ~90+ min | ~500+ | ~$8+ | Research-grade full simulations |

> **Daily default: 20 rounds.** Sufficient for ~170 actions across dual-platform (65 Twitter + 109 Reddit).
> Stop simulation manually if it exceeds 20 rounds (`POST /api/simulation/stop`).

Costs depend on the LLM model used (GPT-4o vs qwen-plus vs local models).

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `LLM_API_KEY` | Yes | OpenAI-compatible API key |
| `LLM_BASE_URL` | Yes | API base URL |
| `LLM_MODEL_NAME` | Yes | Model name (e.g., `gpt-4o`, `qwen-plus`) |
| `ZEP_API_KEY` | Yes | Zep Cloud API key for agent memory |
| `LLM_BOOST_API_KEY` | No | Faster model for high-throughput interactions |
| `LLM_BOOST_BASE_URL` | No | Boost model API URL |
| `LLM_BOOST_MODEL_NAME` | No | Boost model name |
