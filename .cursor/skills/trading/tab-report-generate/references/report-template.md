# Daily Stock Report Template

Standard structure for the daily Korean stock analysis .docx report generated by the backend API.

## Report Sections

| Section | Korean Title | Content |
|---------|-------------|---------|
| Cover | 일일 주식 분석 리포트 | Date, report ID, generation timestamp |
| Executive Summary | 요약 | Market overview, key findings, top signals |
| Market Environment | 시장 환경 분석 | Market breadth, regime classification, risk-on/off assessment |
| Technical Analysis | 기술적 분석 | Per-ticker SMA, RSI, MACD, Stochastic, ADX analysis |
| Turtle Trading Signals | 터틀 트레이딩 시그널 | Donchian breakouts, MA crossovers, position signals |
| Bollinger Bands Analysis | 볼린저 밴드 분석 | Band position, squeeze detection, reversal signals |
| News & Sentiment | 뉴스 및 감성 분석 | Aggregated news, sentiment scores, market-moving events |
| Signal Summary | 시그널 종합 | Buy/sell/neutral table per ticker with confidence levels |
| Risk Warnings | 리스크 경고 | Identified risks, contradicting signals, data quality notes |

## Formatting Standards

| Element | Specification |
|---------|--------------|
| Language | Korean (with English ticker symbols and technical terms) |
| Font | Malgun Gothic (맑은 고딕) for Korean, Arial for English |
| Page size | A4 |
| Date format | YYYY-MM-DD (ISO 8601) |
| Number format | Comma-separated thousands (e.g., 1,234,567) |
| Percentage | One decimal place (e.g., 12.3%) |
| Signal colors | Green (매수), Red (매도), Gray (중립) |

## Data Requirements

Each section requires specific API data refreshes before report generation:

| Section | Required Refresh | Endpoint |
|---------|-----------------|----------|
| Market Environment | Market breadth | `POST /market-breadth/refresh` |
| Technical Analysis | Indicator compute | `POST /technical-analysis/batch-compute` |
| Turtle Signals | Turtle refresh | `POST /turtle/indicators/daily-refresh` |
| Bollinger Analysis | Bollinger refresh | `POST /bollinger-bands/daily-refresh` |
| News & Sentiment | News fetch | `POST /news/fetch` |

## Quality Criteria

- All tracked tickers must appear in the signal summary
- No stale data (all prices must be from the latest trading date)
- Signal recommendations must be consistent with underlying indicator values
- Report date must match the latest trading date, not the calendar date
