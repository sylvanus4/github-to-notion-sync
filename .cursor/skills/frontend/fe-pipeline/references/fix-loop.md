# Fix Loop 규칙

Checkpoint 2 (Design Review) 리포트에서 발견된 이슈를 자동 수정하고 범위 한정 재검증하는 루프.

---

## 1. 이슈 분류

Design Review 리포트의 이슈는 severity 기준으로 처리 순서 결정:

| Severity | 정의 | 자동 수정 | 예시 |
|----------|------|----------|------|
| **Critical** | 기능 불가 또는 심각한 UI 깨짐 | **자동** | tsc 에러, import 누락, TDS Props 오류, i18n 키 누락 |
| **Major** | 기능은 되지만 품질 미달 | **사용자 확인 후** | 간격/색상 불일치, Figma 레이아웃 차이, 접근성 누락 |
| **Minor** | 개선 권장사항 | **스킵** | 코드 정리, 네이밍 개선, 성능 최적화 제안 |

---

## 2. 자동 수정 전략

### 2.1 수정 우선순위

Critical 이슈를 카테고리별로 그룹화하여 순서대로 처리:

```
1순위: TypeScript 에러 (tsc 실패)
  → 타입 수정, import 추가/수정, Props 타입 일치
2순위: TDS 컴포넌트 위반
  → Props 수정 (03-tds-essentials.mdc + 04-tds-detail-catalog.mdc 참조)
  → 잘못된 컴포넌트 교체
3순위: i18n 누락
  → t() 키 추가 (en/ko JSON 동시)
  → 하드코딩 텍스트 → t() 변환
4순위: FSD 구조 위반
  → import 경로 수정, 레이어 의존성 수정
```

### 2.2 수정 단위

한 번에 **하나의 이슈 그룹**만 수정:
- 같은 카테고리의 이슈를 묶어서 처리 (예: "TDS Props 오류 3건" 한꺼번에)
- 서로 다른 카테고리의 이슈는 분리하여 순차 처리

---

## 3. 범위 한정 재검증

수정 후 **전체 재검증이 아닌**, 수정 범위에 해당하는 항목만 재검증:

| 수정 카테고리 | 재검증 범위 |
|-------------|-----------|
| TypeScript 에러 | `npx tsc --noEmit` (수정된 파일 포함 전체, tsc는 부분 실행 불가) |
| TDS 컴포넌트 위반 | 수정된 컴포넌트의 Props 일치 여부만 코드 레벨 확인 |
| i18n 누락 | 수정된 파일의 `t()` 키가 en/ko JSON에 모두 존재하는지 확인 |
| FSD 구조 위반 | 수정된 파일의 import 경로가 레이어 규칙에 맞는지 확인 |
| 레이아웃/시각 차이 | 수정된 컴포넌트를 Figma 스크린샷과 재비교 (browser 비교는 선택) |

---

## 4. 루프 흐름

```
START: Design Review 리포트 수신
  │
  ├─ Critical 이슈 없음 → PASS → 완료
  │
  ├─ Critical 이슈 있음
  │   │
  │   ├─ 1순위 카테고리 그룹 수정
  │   ├─ 범위 한정 재검증
  │   │   ├─ PASS → 다음 카테고리 그룹으로
  │   │   └─ FAIL → 재수정 (retry += 1)
  │   │       ├─ retry < 3 → 같은 이슈 재시도
  │   │       └─ retry >= 3 → 에스컬레이션
  │   │
  │   └─ 모든 Critical 해소
  │       │
  │       ├─ Major 이슈 있음 → 사용자에게 목록 표시
  │       │   ├─ "수정해줘" → Major 수정 루프 진입 (동일 규칙)
  │       │   └─ "괜찮아" → 스킵
  │       │
  │       └─ 최종 tsc 검증 → 완료
```

---

## 5. 최대 재시도 & 에스컬레이션

### 5.1 재시도 한도

| 단위 | 최대 재시도 |
|------|----------|
| 개별 이슈 그룹 | **3회** |
| 전체 Fix Loop | **5회** (모든 카테고리 합산) |

### 5.2 에스컬레이션 처리

3회 실패한 이슈 그룹에 대해:

1. 자동 수정 중단
2. 사용자에게 리포트 전달:

```
⚠️ 자동 수정 실패 (3회 초과):

[Critical] TDS 위반 - TemplateCard.tsx
  시도한 수정: Button variant "danger" → "error" (3회)
  실패 원인: 상위 컴포넌트에서 variant prop을 override하고 있음
  수동 수정 제안: TemplateCard Props에서 variant prop 제거 후 내부에서 직접 지정

남은 Critical 이슈: 0건
남은 Major 이슈: 2건
```

3. 나머지 Critical 이슈는 계속 처리 (실패한 건만 스킵)

---

## 6. 완료 조건

Fix Loop 종료 조건 (어느 하나 충족 시):

| 조건 | 결과 |
|------|------|
| Critical 이슈 모두 해소 + Major 처리 완료/스킵 | **PASS** — 완료 보고 |
| Critical 이슈 모두 해소 + Major는 사용자가 "괜찮아" | **PASS** — 완료 보고 |
| 전체 재시도 5회 초과 | **PARTIAL** — 미해결 이슈 리포트 + 완료 |
| 사용자가 "중단해" | **STOP** — 현재 상태로 완료 |

---

## 7. 수정 시 주의사항

- **수정 범위 최소화**: 이슈와 직접 관련된 코드만 변경. 리팩토링/개선은 금지.
- **기존 로직 보존**: 수정 시 기존 비즈니스 로직이 변경되지 않도록 주의.
- **import 정리**: 수정으로 인한 미사용 import는 함께 정리.
- **i18n 일관성**: en/ko 키를 항상 동시에 추가/수정.
