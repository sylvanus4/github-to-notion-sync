# Phase 0.2 – Input Intake (보유 자료 확인)

Phase 0 완료 후, **사용자 입력에 명시적 URL/경로가 없으면** 보유 자료를 질문한다.
사용자가 최초 메시지에서 Swagger URL, Figma URL, 기획서 경로를 이미 제공한 경우 이 단계를 **스킵**한다.

---

## 질문 항목

| # | 질문 | "있다" 응답 시 | "없다" 응답 시 |
|---|------|-------------|-------------|
| 1 | **API 문서 (Swagger URL)** 가 있나요? | URL 입력 요청 → Phase 1에 전달 | Phase 0.5에서 자동 탐색 → 못 찾으면 Phase 1·4 스킵 |
| 2 | **화면 기획서** 가 있나요? | 경로 입력 요청 → Phase 2 스킵 | Phase 2에서 자동 생성 |
| 3 | **Figma URL** 이 있나요? | URL 입력 요청 → Phase 3 실행 | Phase 3 스킵 → 유사 화면 레이아웃 참조 |

## 표시 형식

```
━━━ 보유 자료 확인 ━━━

구현에 필요한 자료가 있는지 확인합니다. 없으면 자동으로 생성/탐색합니다.

1. 📡 API 문서 (Swagger URL)가 있나요?
2. 📄 화면 기획서가 있나요?
3. 🎨 Figma URL이 있나요?

→ 있는 항목만 URL/경로를 알려주세요. 없으면 "없다"라고 하시면 됩니다.
```

## 스킵 조건

| 조건 | 동작 |
|------|------|
| 최초 메시지에 Swagger/Figma URL 포함 | Intake 스킵 → 해당 URL 직접 사용 |
| 최초 메시지에 기획서 경로 포함 | Intake 스킵 → 해당 경로 직접 사용 |
| Phase 0에서 모든 산출물 존재 확인 | Intake 스킵 → 바로 미완료 Phase 실행 |

## Intake 결과 → Phase 분기

| API | 기획서 | Figma | 다음 단계 |
|-----|-------|-------|----------|
| 없다 | 없다 | 없다 | Phase 0.5(전체 발견) → 자동 생성 |
| URL 제공 | 없다 | 없다 | Phase 0.5(도메인·화면만) → Phase 1 → 2(자동) |
| URL 제공 | 경로 제공 | 없다 | Phase 1 → 3.5 → CP1 |
| URL 제공 | 경로 제공 | URL 제공 | Phase 1 → 3 → 3.5 → CP1 |
| 없다 | 경로 제공 | URL 제공 | Phase 0.5(API 발견) → 3 → 3.5 → CP1 |
