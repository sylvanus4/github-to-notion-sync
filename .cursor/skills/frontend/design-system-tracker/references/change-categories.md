# Design System Change Categories

디자인 시스템 변경사항 분류 체계.

## 변경 유형

### 1. 컴포넌트 변경

| 변경 유형 | 영향 범위 | 기본 영향도 |
|-----------|----------|------------|
| 컴포넌트 추가 | 해당 컴포넌트만 | Low |
| 컴포넌트 삭제/deprecated | 사용처 전체 | Critical |
| variant 추가 | 해당 컴포넌트 | Medium |
| variant 삭제 | 해당 variant 사용처 | High |
| props 변경 | 해당 컴포넌트 사용처 | High |
| 기본값 변경 | 명시적 설정 없는 사용처 | Medium |
| 레이아웃 변경 | 해당 컴포넌트 + 컨테이너 | Medium |

### 2. 토큰 변경

| 변경 유형 | 영향 범위 | 기본 영향도 |
|-----------|----------|------------|
| 색상 토큰 값 변경 | 참조하는 모든 컴포넌트 | High~Critical |
| 타이포그래피 변경 | 텍스트 요소 전체 | High |
| Spacing 변경 | 레이아웃 전체 | Medium |
| Border radius 변경 | 해당 요소 | Low |
| Shadow 변경 | 해당 요소 | Low |
| 토큰 이름 변경 | 코드 참조 전체 | Critical (breaking) |

### 3. 스타일 변경

| 변경 유형 | 영향 범위 | 기본 영향도 |
|-----------|----------|------------|
| Fill 스타일 추가/변경 | 참조 요소 | Medium |
| Text 스타일 변경 | 참조 텍스트 | Medium |
| Effect 스타일 변경 | 참조 요소 | Low |
| Grid 스타일 변경 | 레이아웃 | Medium |

## 영향도 에스컬레이션 규칙

기본 영향도에서 상향 조정하는 기준:

- 사용처 10개 이상 → 한 단계 상향
- 핵심 플로우 (결제, 가입, 로그인) 영향 → 한 단계 상향
- Breaking change (이름 변경, 삭제) → Critical 고정
- 접근성 영향 (color contrast, font size) → 한 단계 상향

## 기획 문서 대조 우선순위

기획서-디자인 대조 시 우선 확인 항목:

1. **상태 정의 일치** — 기획서에 명시된 상태가 디자인에 모두 존재하는지
   - loading, error, empty, success, disabled 등
2. **인터랙션 패턴** — 클릭, 호버, 포커스 동작이 기획과 일치하는지
3. **반응형 동작** — 브레이크포인트별 동작 정의 일치
4. **접근성 요소** — 기획서의 접근성 요구사항이 디자인에 반영되었는지
