# PRD ↔ 디자인 시스템 대조 체크리스트

`design-system-tracker` **Impact Analysis** 단계에서 PRD가 제공됐을 때 사용한다.

## 컴포넌트·패턴

- [ ] PRD에 나온 주요 UI 블록이 DS에 컴포넌트(또는 variant)로 존재하는가?
- [ ] 상태(로딩/빈/에러/비활성)가 PRD와 DS 모두에 정의돼 있는가?
- [ ] 인터랙션(모달, 토스트, 드로어 등) 패턴이 PRD와 DS 용어·행동이 일치하는가?

## 토큰·스타일

- [ ] PRD 또는 스펙에 명시된 색/타이포/간격이 DS 토큰과 매핑 가능한가?
- [ ] 브레이킹 변경(색상·radius 등)이 PRD의 시각 요구와 충돌하지 않는가?

## 문서·추적

- [ ] 불일치 항목마다 **기획 수정 / 디자인 수정 / 구현 확인** 중 권장 조치를 적었는가?
- [ ] 근거(프레임·컴포넌트명, PRD 섹션)를 리포트에 남겼는가?
