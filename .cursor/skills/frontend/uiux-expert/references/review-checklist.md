## 1. 시각적 계층 구조 (Visual Hierarchy)

- [ ] 화면에 진입 후 3초 이내에 주요 액션을 식별할 수 있는가
- [ ] 제목 → 본문 → 보조 정보의 계층이 명확한가
- [ ] CTA(Call to Action) 버튼이 시각적으로 가장 눈에 띄는가
- [ ] 정보 그룹핑이 논리적이고 시각적으로 구분되는가

| 심각도 | 기준 |
|--------|------|
| Critical | 주요 액션이 3초 내 식별 불가 |
| Major | 계층이 존재하나 2차 요소와 구분이 약함 |
| Minor | 미세한 계층 개선 여지 |

## 2. 간격 및 리듬 (Spacing & Rhythm)

- [ ] 모든 간격이 디자인 토큰(4px 배수 등)을 사용하는가
- [ ] 관련 요소 간 근접성(proximity) 원칙이 적용되는가
- [ ] 섹션 간 간격이 일관되는가
- [ ] 콘텐츠 영역에 충분한 여백(breathing room)이 있는가

| 심각도 | 기준 |
|--------|------|
| Critical | 하드코딩 px/rem으로 토큰 미사용 |
| Major | 토큰 사용하나 간격 불일치 3건 이상 |
| Minor | 간격이 적절하나 최적은 아님 |

## 3. 타이포그래피 (Typography)

- [ ] 폰트 사이즈가 디자인 토큰으로 정의되는가
- [ ] 제목-본문-캡션의 크기 계층이 명확한가
- [ ] 줄 높이(line-height)가 가독성에 적합한가
- [ ] 폰트 굵기로 강조가 적절히 표현되는가

| 심각도 | 기준 |
|--------|------|
| Critical | 토큰 외 임의 폰트 사이즈 사용 |
| Major | 계층 구분이 어려움 (크기 차이 2px 이하) |
| Minor | 가독성 문제 없으나 최적화 여지 |

## 4. 컴포넌트 일관성 (Component Consistency)

- [ ] 동일 기능에 동일 **Radix + Tailwind** 패턴을 사용하는가 (`design-system.mdc`)
- [ ] variant(버튼·배지 등)가 올바르게 적용되는가
- [ ] 문서화된 패턴이 있는데 임의 커스텀으로 우회하지 않는가
- [ ] 컴포넌트 사용이 `design-system.mdc` 가이드와 일치하는가

| 심각도 | 기준 |
|--------|------|
| Critical | 문서화된 패턴이 있는데 불필요하게 직접 구현 |
| Critical | 동일 기능에 다른 컴포넌트 사용 |
| Major | variant 오용 (예: primary 대신 secondary) |
| Minor | 가이드라인 내 권장 사용법과 미세한 차이 |

## 5. 상태 처리 (State Handling)

- [ ] 로딩 상태(skeleton/spinner)가 구현되는가
- [ ] 빈 상태(empty state)에 안내 메시지 + CTA가 있는가
- [ ] 에러 상태에 원인 설명 + 복구 액션이 있는가
- [ ] 성공/실패 피드백(toast/alert)이 있는가
- [ ] disabled 상태에 tooltip으로 이유를 알려주는가

| 심각도 | 기준 |
|--------|------|
| Critical | 로딩/에러/빈 상태 중 하나라도 누락 |
| Major | 상태는 있으나 안내 부족 |
| Minor | 상태 처리 있으나 UX 개선 여지 |

## 6. 접근성 (Accessibility)

- [ ] 색상 대비가 WCAG 2.1 AA 기준을 충족하는가 (4.5:1 텍스트, 3:1 UI)
- [ ] 키보드만으로 모든 기능에 접근 가능한가
- [ ] 포커스 상태가 시각적으로 구분되는가
- [ ] 스크린 리더용 ARIA 라벨이 적절한가
- [ ] 색상만으로 정보를 전달하지 않는가 (아이콘/텍스트 병행)

| 심각도 | 기준 |
|--------|------|
| Critical | WCAG AA 대비 미충족 |
| Major | 키보드 접근 불가 기능 존재 |
| Major | ARIA 라벨 누락 |
| Minor | 포커스 링 스타일 개선 여지 |

## 7. 반응형 (Responsiveness)

- [ ] Desktop (1280px+) 레이아웃이 적절한가
- [ ] Tablet (768-1279px) 레이아웃이 적절한가
- [ ] Mobile (< 768px) 레이아웃이 적절한가 (해당 시)
- [ ] 텍스트 truncation이 적절히 처리되는가
- [ ] 테이블/리스트의 가로 스크롤이 자연스러운가

| 심각도 | 기준 |
|--------|------|
| Critical | 주요 뷰포트에서 레이아웃 깨짐 |
| Major | 콘텐츠 잘림/오버플로우 |
| Minor | 특정 뷰포트에서 최적은 아님 |

## 8. 인터랙션 패턴 (Interaction Patterns)

- [ ] 삭제 등 위험 동작에 확인 다이얼로그가 있는가
- [ ] 폼 유효성 검사가 실시간으로 피드백되는가
- [ ] 클릭 가능한 요소에 hover/active 상태가 있는가
- [ ] 드래그, 정렬 등의 동작이 예측 가능한가
- [ ] 페이지 이동 시 데이터 손실 방지(unsaved changes) 처리가 있는가

| 심각도 | 기준 |
|--------|------|
| Critical | 위험 동작에 확인 없이 즉시 실행 |
| Major | 유효성 검사 없이 서버 요청 |
| Major | hover/active 상태 누락 |
| Minor | 인터랙션이 작동하나 피드백이 약함 |

## Nielsen 10 Heuristics 평가 기준

각 항목을 1-5점으로 평가 (1: 심각한 문제, 5: 우수).

| # | 휴리스틱 | 평가 관점 |
|---|---------|-----------|
| H1 | 시스템 상태의 가시성 | 현재 상태, 진행률, 로딩이 명확히 표시되는가 |
| H2 | 시스템과 현실 세계의 일치 | 용어, 아이콘, 메타포가 사용자에게 자연스러운가 |
| H3 | 사용자 제어와 자유 | 실행 취소, 뒤로 가기, 취소가 항상 가능한가 |
| H4 | 일관성과 표준 | 플랫폼 컨벤션과 내부 패턴이 일관되는가 |
| H5 | 오류 예방 | 실수를 유발하는 UI 요소가 없는가 |
| H6 | 인식 > 회상 | 옵션/액션이 보이는가, 암기가 필요 없는가 |
| H7 | 유연성과 효율성 | 초보자와 전문가 모두 효율적으로 사용 가능한가 |
| H8 | 미학과 미니멀 디자인 | 불필요한 정보 없이 핵심만 표시하는가 |
| H9 | 오류 복구 지원 | 에러 메시지가 원인과 해결 방법을 안내하는가 |
| H10 | 도움말과 문서 | 필요 시 도움말에 접근 가능한가 |

**점수 해석:**
- 4.0+ 평균: 우수 — Minor 개선만 필요
- 3.0-3.9 평균: 양호 — Major 이슈 존재
- 2.9 이하: 미흡 — Critical 이슈 즉시 대응 필요
