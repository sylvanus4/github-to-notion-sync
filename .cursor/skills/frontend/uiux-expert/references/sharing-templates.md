## Notion 페이지 템플릿

### 페이지 구조

페이지 제목: `[UI/UX 리뷰] {스코프/영역} — {YYYY-MM-DD}`

> 리뷰 일자: {YYYY-MM-DD}
> 리뷰어: AI Assistant + {담당자명}
> 판정: **{PASS / FAIL}**

## Slack 메시지 템플릿

### 메인 메시지 (`#기획-자동화`)

```
:art: *[UI/UX 리뷰 완료]* {스코프/영역} — {YYYY-MM-DD}

*판정:* {PASS :white_check_mark: / FAIL :x:}
*Critical:* {N}건 | *Major:* {N}건 | *Minor:* {N}건
*휴리스틱 평균:* {N.N}/5.0
*일관성 등급:* {A/B/C/D}

:notion: 상세 리포트: {Notion 페이지 링크}
```

### 쓰레드 — 진행 상황

```
:clipboard: *진행 상황 요약*

*{기능/영역 A}*
• 완료: {완료 항목 요약}
• 진행 중: {진행 항목 요약}
• 블로커: {블로커 요약 또는 "없음"}

*{기능/영역 B}*
• (동일 구조)
```

### 쓰레드 — Critical 이슈 (있을 때만)

```
:rotating_light: *Critical 이슈 상위 {N}건*

1. *{이슈 제목}* — {화면명}
   현재: {현재 상태}
   권고: {수정 방향}

2. (동일 구조)
```

### 쓰레드 — 일관성 (불일치 있을 때만)

```
:scales: *기능 간 일관성 불일치*

| 항목 | 차이 | 심각도 |
|------|------|--------|
| {패턴명} | {차이 요약} | {Critical/Major} |
```

### 쓰레드 — 전주 대비 트렌드

```
:chart_with_upwards_trend: *전주 대비 변화*

Critical: {N} → {N} ({변화})
Major: {N} → {N} ({변화})
휴리스틱: {N.N} → {N.N} ({변화})
```

## 정기 공유 스케줄 가이드

| 주기 | 범위 | 권장 모듈 |
|------|------|----------|
| 주간 | 해당 주 변경 화면 중심 | `--progress --quality --share` |
| 격주 | 전체 화면 + 기능 간 비교 | `--all` |
| 월간 | 전체 리뷰 + 트렌드 분석 + 정책 업데이트 | `--all` + 정책 문서 갱신 |

## Notion 페이지 생성 가이드

### MCP 호출 패턴

1. 상위 페이지 ID 확인 (사용자 지정 또는 기본 DB)
2. Notion MCP `create_page`로 페이지 생성
3. 리포트 내용을 HTML 블록으로 변환 (파이프 테이블 → `<table>`)
4. 15KB 초과 시 하위 페이지로 분할

### 제목 규칙

`[UI/UX 리뷰] {스코프/영역} — {YYYY-MM-DD}` (출력 표준 준수)
