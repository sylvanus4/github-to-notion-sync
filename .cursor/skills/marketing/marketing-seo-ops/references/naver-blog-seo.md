## 1. 네이버 C-Rank

C-Rank는 블로그 채널의 신뢰도를 측정하는 알고리즘이다.

| 요소 | 설명 |
|------|------|
| 문서 품질 | 독창적 내용, 정보 깊이 |
| 채널 전문성 | 특정 주제 집중도 (1-2개 분야) |
| 독자 반응 | 체류 시간, 공감, 댓글 |
| 업데이트 빈도 | 정기적 발행 |

핵심: 잡다한 주제보다 전문 분야 집중이 C-Rank에 유리하다.

---

## 2. D.I.A.+ 알고리즘

문서 단위의 품질 평가 지표:

- **D (Definiteness)**: 정보의 구체성과 명확성
- **I (Information)**: 독자에게 실질적 가치 제공
- **A (Accuracy)**: 정확한 사실 기반
- **+ (Originality)**: 직접 경험 + 유일한 관점

## 3. 키워드 전략

### 리서치 프로세스

1. 네이버 검색창 자동완성어 수집
2. 네이버 데이터랩에서 검색량 추이 확인
3. 롱테일 키워드 (3단어 이상) 우선 공략
4. 경쟁 글 확인 후 차별화 포인트 설정

### 키워드 배치 원칙

- 제목에 핵심 키워드 포함 (앞쪽 배치)
- 소제목에 연관 키워드 자연스럽게 배치
- 본문 첫 100자에 핵심 키워드 1회
- 키워드 밀도 1-2% 유지 (과다 반복 금지 -- 저품질 처리됨)

## 4. 글 구조 최적화

### 권장 구성

```
제목: [핵심 키워드] + 구체적 혜택 (30자 이내)
서론: 독자의 문제 공감 (150-200자)
본문: 소제목으로 분리된 3-5개 섹션
  - 각 섹션: 소제목 + 본문 300-500자
  - 이미지: 섹션당 1-2장 (직접 촬영 우선)
  - 목록형 정리: 핵심 내용은 글머리 기호 활용
결론: 요약 + 행동 유도 문구
```

### 콘텐츠 품질 체크리스트

- [ ] 1,500자 이상 (정보성 글 기준)
- [ ] 직접 촬영 이미지 최소 3장
- [ ] 외부 링크보다 내부 링크 우선
- [ ] 출처 명시 (인용 시)
- [ ] 발행 후 48시간 내 수정 최소화

## 5. GEO (Generative Engine Optimization)

2026년 AI 검색 시대에 대응하는 최적화 전략.

### SEO vs GEO 차이

| 비교 항목 | 전통 SEO | GEO |
|-----------|---------|-----|
| 최적화 대상 | 검색 엔진 크롤러 | AI 언어 모델 |
| 성공 지표 | 클릭률(CTR), 순위 | AI 인용 빈도, 브랜드 언급 |
| 콘텐츠 형식 | 키워드 중심 | 질문-답변 구조 |
| 링크의 역할 | 핵심 순위 요인 | 신뢰도 보조 지표 |
| 결과 표시 | 링크 목록 | 요약 답변 + 출처 |

### AI 인용을 위한 콘텐츠 원칙

**구조화된 답변 작성**
- 질문 -> 직접 답변 -> 근거 -> 예시 순서
- FAQ 섹션 반드시 포함 (AI가 직접 인용)
- 정의형 단락: "X란 Y를 의미합니다. 구체적으로는 Z입니다" 패턴

**신뢰성 신호 강화**
- 작성자 전문성 표기 (이름, 자격, 경험)
- 발행일 및 업데이트 날짜 명시
- 통계/수치에 출처 인용
- 전문 기관 또는 연구 결과 인용

**Schema.org 마크업 적용**
- FAQ 스키마: AI Overview 인용 가능성 3배 상승
- How-To 스키마: 단계별 안내 콘텐츠에 필수
- Article 스키마: 작성자 신뢰도 전달

### 네이버 AI 검색 최적화

- C-Rank + D.I.A.+ 기반 신뢰 블로그 우선 인용
- 명확한 정의, 수치, 절차가 포함된 글이 유리
- 네이버 지식IN 형식(질문-답변)과 유사한 구조 권장

### 구글 AI Overview(SGE) 최적화

- E-E-A-T 기준 충족이 AI 인용의 선행 조건
- 길이보다 답변의 명확성이 중요
- "직접 답변 + 예외사항 + 추가 정보" 3단 구조 권장

## 6. SEO + GEO 동시 최적화 체크리스트

- [ ] 주요 쿼리를 질문형 소제목으로 변환 (H2: "X란 무엇인가?")
- [ ] 각 섹션 첫 문장에 직접 답변 배치
- [ ] FAQ 섹션 최소 5개 이상
- [ ] 통계/수치 2026년 기준 데이터 사용
- [ ] Schema.org 마크업 적용 (FAQ, HowTo, Article)
- [ ] 작성자 bio 페이지 및 소셜 프로필 연동
- [ ] 시리즈 기능 활용으로 연관 글 묶기 (C-Rank 향상)

### 피해야 할 패턴

- AI 생성 콘텐츠 대량 발행 (구글/네이버 모두 패널티)
- 키워드 채우기식 글 (GEO에서도 저평가)
- 업데이트 없이 방치된 오래된 글
