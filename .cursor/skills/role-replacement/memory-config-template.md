# MemKraft Memory Configuration Template for Role Replacement Skills

> Reference template for integrating MemKraft memory into role-replacement thin harness skills.
> Each role skill must define Pre-Load queries and Write-Back schemas.

## Memory Tiers

| Tier | Store | Scope | TTL |
|------|-------|-------|-----|
| HOT | MemKraft session | Current dispatch cycle | Until next dispatch |
| WARM | MemKraft topics | Cross-session patterns | Weekly decay via Dream Cycle |
| Knowledge | LLM Wiki (KB) | Persistent compiled knowledge | Compiled articles, no TTL |

## Pre-Load Template

Before any phase execution, query `ai-context-router`:

```
PERSONAL (MemKraft HOT):
  - Yesterday's {role} execution summary
  - Pending items from previous dispatch
  - Active tracking items (deals, tickets, synergies)
  - Recent errors and recovery actions
  - User preference overrides

PERSONAL (MemKraft WARM):
  - Weekly performance trends for this role
  - Recurring patterns detected by Dream Cycle
  - Automation level status and history
  - User corrections and preference evolution

KNOWLEDGE (LLM Wiki):
  - Role-specific architecture documentation
  - Composed skill configurations
  - Error classification guides
  - Operational policies and constraints
```

## Write-Back Template

After dispatch completes, persist state:

```
MemKraft HOT (always):
  - Today's dispatch manifest path
  - Key metrics (items processed, errors, decisions)
  - New tracking items created
  - Phase guard reuse count
  - Errors encountered and severity

MemKraft WARM (weekly or on significant events):
  - Performance trend delta
  - Automation level review candidates
  - New patterns detected
  - Circuit breaker state changes
```

## Integration Pattern (Pseudocode)

```python
# Phase 0: Pre-Load
context = ai_context_router.query(
    personal_queries=["yesterday's {role} state", "pending items"],
    knowledge_queries=["{role} architecture", "error classification"],
    provenance_separate=True
)

# ... execute phases ...

# Phase N: Write-Back
memkraft.write_hot({
    "dispatch_date": today,
    "manifest": manifest_path,
    "metrics": { "processed": count, "errors": error_count },
    "tracking": new_items,
    "guard_reuse": reuse_count
})

if is_friday or significant_event:
    memkraft.write_warm({
        "trend_delta": compute_delta(this_week, last_week),
        "automation_review": candidates,
        "new_patterns": detected_patterns
    })
```

## Provenance Tags

All memory entries must include provenance:

| Tag | Meaning |
|-----|---------|
| `source: dispatch/{role}` | Generated by role dispatcher |
| `source: axis/{N}` | Generated by a specific axis |
| `source: user-correction` | User manually overrode or corrected |
| `source: dream-cycle` | Consolidated during nightly Dream Cycle |
| `confidence: high/medium/low` | Confidence in the data point |
| `timestamp: ISO-8601` | When the entry was created |
