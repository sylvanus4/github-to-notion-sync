# Remotion Brownfield Installation Guide

Reference for setting up Remotion in the existing project. This guide covers the exact steps for Phase 2 (Remotion Project Scaffolding) of the remotion-motion-forge skill.

## Prerequisites

| Tool | Min Version | Verify |
|------|-------------|--------|
| Node.js | 18+ | `node --version` |
| npm or pnpm | any | `npm --version` |
| ffmpeg | any | `ffmpeg -version` |

## Step 1: Create the Remotion Directory

```bash
mkdir -p remotion/tokens remotion/compositions remotion/components remotion/public
```

## Step 2: Initialize package.json

Create `remotion/package.json`:

```json
{
  "name": "motion-forge-remotion",
  "private": true,
  "scripts": {
    "studio": "remotion studio",
    "render": "remotion render",
    "compositions": "remotion compositions"
  },
  "dependencies": {
    "remotion": "^4.0.0",
    "@remotion/cli": "^4.0.0",
    "@remotion/player": "^4.0.0",
    "react": "^18.0.0",
    "react-dom": "^18.0.0"
  },
  "devDependencies": {
    "typescript": "^5.0.0",
    "@types/react": "^18.0.0"
  }
}
```

## Step 3: Configure TypeScript

Create `remotion/tsconfig.json`:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "jsx": "react-jsx",
    "strict": true,
    "esModuleInterop": true,
    "outDir": "dist",
    "baseUrl": ".",
    "paths": {
      "@tokens/*": ["tokens/*"],
      "@components/*": ["components/*"]
    }
  },
  "include": ["**/*.ts", "**/*.tsx"]
}
```

## Step 4: Create the Entry Point

Create `remotion/index.ts`:

```typescript
import { registerRoot } from "remotion";
import { RemotionRoot } from "./Root";

registerRoot(RemotionRoot);
```

`registerRoot()` tells Remotion where to find the composition registry. This file is the entry point for both `remotion studio` (preview) and `remotion render` (CLI rendering).

## Step 5: Create the Root Component

Create `remotion/Root.tsx`:

```tsx
import React from "react";
import { Composition } from "remotion";

export const RemotionRoot: React.FC = () => {
  return (
    <>
      {/* Compositions are registered here by Phase 3.
          Each <Composition> defines an independently renderable video. */}
    </>
  );
};
```

Compositions are added here during Phase 3. Each `<Composition>` entry defines:
- `id` — unique identifier used in render commands
- `component` — the React component that renders the video
- `durationInFrames` — total length (frames = seconds × fps)
- `fps` — frames per second (default: 30)
- `width` / `height` — resolution in pixels

Example registration (added during Phase 3):

```tsx
<Composition
  id="PromoReel"
  component={PromoReelComposition}
  durationInFrames={30 * 30}
  fps={30}
  width={1920}
  height={1080}
/>
```

## Step 6: Install Dependencies

```bash
cd remotion && npm install
```

## Step 7: Verify Installation

```bash
cd remotion && npx remotion compositions index.ts
```

Expected output: an empty list (no compositions registered yet) with no errors. If you see module resolution errors, check that `react` and `remotion` installed correctly.

## Rendering Commands

Once compositions are registered in Phase 3:

```bash
# Render a specific composition to MP4
cd remotion && npx remotion render index.ts <CompositionId> ../outputs/motion-forge/<date>/<name>.mp4 --codec h264

# Preview in browser (Remotion Studio)
cd remotion && npx remotion studio

# List registered compositions
cd remotion && npx remotion compositions index.ts

# Draft render at half resolution (faster)
cd remotion && npx remotion render index.ts <CompositionId> output.mp4 --scale 0.5

# Verbose logging for debugging
cd remotion && npx remotion render index.ts <CompositionId> output.mp4 --log=verbose
```

## Directory Structure After Setup

```
remotion/
├── index.ts                    # Entry point — registerRoot()
├── Root.tsx                    # Composition registry
├── package.json                # Remotion + React dependencies
├── tsconfig.json               # TypeScript config with path aliases
├── tokens/
│   └── design-tokens.ts        # Generated by Phase 1
├── compositions/               # Video compositions (Phase 3)
├── components/                 # Reusable animation components (Phase 3)
└── public/                     # Static assets (images, logos)
```

## Updating Remotion

```bash
cd remotion && npx remotion upgrade
```

This updates all `@remotion/*` packages to compatible versions.

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `Cannot find module 'remotion'` | Run `cd remotion && npm install` |
| `No compositions found` | Ensure `Root.tsx` has `<Composition>` entries and `index.ts` calls `registerRoot` |
| Studio opens but shows blank | Check browser console for errors; ensure React is installed |
| `ERR_MODULE_NOT_FOUND` | Verify `tsconfig.json` has `"moduleResolution": "bundler"` |
| Render hangs | Add `--timeout 120000` flag (2 min timeout) |
