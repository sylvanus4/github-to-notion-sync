## 작성 프로세스

### Phase 1: 사전 조사

1. 지원 기관/사업 특정 (NRF 신진연구, IITP AI 기반기술 등)
2. 공고문에서 평가 기준 및 배점 추출
3. 관련 선행 연구 및 기존 수행 과제 파악

### Phase 2: 핵심 섹션 작성

**연구 개요**
- 연구 목적 + 필요성 (국내외 현황 대비 연구 공백)
- 최종 결과물 명세 (정량 목표: 정확도 %, 처리 속도, 논문 수)
- TRL (Technology Readiness Level) 시작/종료 단계 명시

**연구 내용 및 방법**
- 세부 연구 내용 (연차별 구분)
- 핵심 기술 요소 및 혁신점
- 연구 방법론 (실험 설계, 데이터셋, 평가 지표)

**연구 역량**
- 연구책임자 이력 (관련 논문, 특허, 프로젝트)
- 참여 연구원 역할 및 기여도
- 연구 인프라 (장비, 시설, 협력 기관)

**연구개발비 산정**
- 직접비: 인건비(연구원급), 연구시설장비비, 연구재료비, 위탁연구개발비
- 간접비: 기관 간접비율 적용
- 연차별 집행 계획

### Phase 3: 시각화 및 정리

- 연구 흐름도 (Mermaid flowchart 또는 Draw.io)
- 간트차트 (연차별 마일스톤)
- 핵심 성과 지표 표

### Phase 4: 품질 검증

- 평가 기준별 자가 점검
- 정량 목표 달성 가능성 현실성 검토
- 예산 산정 근거 적정성
- 선행 연구 인용 적절성

## 기관별 특이사항

### NRF (한국연구재단)
- IRIS 시스템 통해 온라인 접수
- 연구윤리 확인서 필수
- 인문사회/자연과학/공학 분야별 양식 상이
- 신진연구: PI 경력 7년 이내 조건

### IITP (정보통신기획평가원)
- 과제 기획 보고서 형식 (정해진 양식)
- 기업 참여 시 대응자금 매칭 필요
- SW/AI 특화: 데이터셋, 벤치마크 성능 목표 중시

### KIAT (산업기술평가관리원)
- 산업 현장 수요 연계 강조
- 참여기업 사업화 계획 필수
- 대학-기업 공동 연구 가산점

---

## 공통 유의사항

- **마감 시한 엄수**: IRIS/과제관리시스템 마감 당일 오류 빈발
- **중복 과제 금지**: 동일/유사 내용 타 기관 동시 수행 불가
- **정량 목표 구속력**: 제안한 목표는 연차 평가에서 달성 여부 검증
- **인건비 기준**: 정부 인건비 테이블(연구원급) 준수

## Harness: 연구비 신청서 팀 구성 (5-Agent)

복잡한 연구비 신청 시 아래 역할의 전문가 에이전트를 병렬/순차 실행:

| 에이전트 | 역할 | 산출물 |
|---------|------|--------|
| **공고 분석관** | 공고문 파싱, 평가 기준/배점 추출, 자격 적합성 판단 | 공고 분석 시트 |
| **연구 기획관** | 선행 연구 조사, 연구 방법론 설계, TRL 설정 | 연구 개요 초안 |
| **사업 기획관** | 사업화 전략, 시장 분석, 수익 모델 | 사업화 계획서 |
| **예산 설계관** | 비목별 예산 산정, 근거 작성, 연차별 집행 계획 | 예산서 |
| **신청서 검증관** | 전체 일관성 검토, 평가 기준 매핑, 오류 탐지 | 검증 리포트 |

### 실행 순서

```
Phase 1 (병렬): 공고 분석관 + 연구 기획관
Phase 2 (순차): 사업 기획관 -> 예산 설계관
Phase 3 (순차): 신청서 검증관 -> 피드백 반영 -> 최종 산출물
```
