# Requirements Interview Framework

구조화된 질문으로 요구사항을 수집하는 프레임워크.

## 질문 카테고리

### 1. 기능 요구사항 (필수)

| 질문 | 목적 | 예시 답변 |
|------|------|----------|
| 이 시스템의 핵심 기능 3-5가지는? | 범위 파악 | "사용자 등록, 상품 검색, 결제" |
| 각 기능에서 사용자의 주요 플로우는? | 동작 이해 | "검색 → 상품 상세 → 장바구니 → 결제" |
| 어떤 데이터를 입력하고 어떤 결과를 보여주나? | I/O 정의 | "키워드 입력 → 상품 리스트 (이미지, 가격, 평점)" |
| 실시간으로 처리해야 하는 기능이 있나? | 아키텍처 방향 | "주문 현황 실시간 업데이트" |

### 2. 규모 및 성능 (필수)

| 질문 | 목적 | 업계 기본값 (미응답 시) |
|------|------|---------------------|
| 예상 일간 활성 사용자(DAU)는? | 인프라 규모 | 1,000 DAU |
| 최대 동시 접속자는? | 피크 처리 | DAU × 0.1 |
| 허용 가능한 응답 시간은? | 성능 기준 | 페이지 2초, API 500ms |
| 하루 데이터 발생량은? | 스토리지 | 1GB/일 |
| 데이터 보관 기간은? | 스토리지 + 규정 | 1년 |

### 3. 보안 및 규정 (선택)

| 질문 | 목적 |
|------|------|
| 개인정보를 처리하나? 어떤 종류? | 규정 준수 범위 |
| 결제 기능이 있나? | PCI DSS 준수 필요성 |
| 특정 지역 규정을 따라야 하나? (GDPR 등) | 리전 선택 |
| 내부 보안 가이드라인이 있나? | 보안 제약 |

### 4. 연동 및 의존성 (선택)

| 질문 | 목적 |
|------|------|
| 연동해야 하는 기존 시스템이 있나? | 통합 복잡도 |
| 외부 API를 사용해야 하나? | 의존성 리스크 |
| 데이터를 다른 시스템에서 가져와야 하나? | 데이터 파이프라인 |

### 5. 비용 및 일정 (선택)

| 질문 | 목적 |
|------|------|
| 월 인프라 예산 범위는? | 비용 제약 |
| 목표 출시일은? | 일정 제약 |
| 단계적 출시가 가능한가? | 스코프 조절 |

## 질문 전략

1. **기획안에서 먼저 추출**: 질문 전에 기획안에서 답을 추출할 수 있는지 확인
2. **한 번에 3개 이하**: 질문 피로 방지
3. **구체적 예시 포함**: "사용자 규모는?" 대신 "예상 DAU는? (예: 1,000명, 10,000명, 100,000명)"
4. **기본값 제시**: "미정이면 일반적인 B2B SaaS 기준 (DAU 1,000)으로 분석하겠습니다"
5. **필수와 선택 구분**: 필수 항목이 모두 확보되면 선택 항목은 기본값 적용
