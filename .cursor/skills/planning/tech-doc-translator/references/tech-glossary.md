# Tech Glossary for Non-Technical Audiences

비기술자를 위한 기술 용어 사전. 기획·디자인 관점의 설명 포함.

## 인프라 / 아키텍처

| 용어 | 기획자용 설명 | 기획 영향 |
|------|-------------|----------|
| **API** | 서비스 간 데이터를 주고받는 통로 | 외부 서비스 연동, 데이터 연계 가능 범위 결정 |
| **마이크로서비스** | 하나의 큰 시스템을 독립적인 작은 서비스로 분리한 구조 | 기능별 독립 배포 가능, 팀별 독립 개발 |
| **CDN** | 전 세계에 콘텐츠를 미리 배포해두는 네트워크 | 해외 사용자 로딩 속도 개선 |
| **로드밸런서** | 사용자 요청을 여러 서버로 분산하는 장치 | 동시 접속자 처리 능력 결정 |
| **캐시** | 자주 쓰는 데이터를 빠르게 접근할 수 있는 임시 저장소 | 응답 속도 개선, 데이터 최신성 트레이드오프 |

## 데이터

| 용어 | 기획자용 설명 | 기획 영향 |
|------|-------------|----------|
| **스키마** | 데이터의 구조 정의 (어떤 항목이 어떤 형식으로 저장되는지) | 입력 폼 필드, 필수/선택 항목 결정 |
| **마이그레이션** | 기존 데이터를 새 구조로 옮기는 작업 | 일정 영향, 다운타임 가능성 |
| **인덱스** | 데이터 검색 속도를 높이기 위한 목차 | 검색/필터 기능 성능 |
| **트랜잭션** | 여러 작업을 하나로 묶어 전부 성공 아니면 전부 취소 | 결제, 주문 등 정합성 필요 기능 |

## 인증 / 보안

| 용어 | 기획자용 설명 | 기획 영향 |
|------|-------------|----------|
| **OAuth** | 다른 서비스 계정으로 로그인하는 표준 방식 | 소셜 로그인 기능 |
| **JWT** | 로그인 상태를 증명하는 디지털 신분증 | 세션 만료 정책, 자동 로그인 |
| **RBAC** | 역할(관리자, 일반 등)에 따라 접근 권한을 다르게 설정 | 권한 체계 설계 |
| **암호화** | 데이터를 읽을 수 없는 형태로 변환하여 보호 | 개인정보 보호, 규정 준수 |
| **Rate Limiting** | 일정 시간 내 요청 수를 제한하는 보호 장치 | 무료/유료 플랜 차등, 남용 방지 |

## 클라우드

| 용어 | 기획자용 설명 | 기획 영향 |
|------|-------------|----------|
| **SaaS** | 웹 브라우저로 바로 쓰는 소프트웨어 | 설치 불필요, 구독 모델 |
| **IaaS** | 가상 서버를 빌려 쓰는 서비스 | 인프라 비용, 확장성 |
| **서버리스** | 서버 관리 없이 코드만 실행하는 방식 | 사용량 기반 과금, 자동 확장 |
| **컨테이너** | 앱을 독립적인 환경에서 실행하는 패키징 기술 | 환경 일관성, 배포 속도 |
| **Auto Scaling** | 사용량에 따라 서버가 자동으로 늘거나 줄어드는 기능 | 트래픽 급증 대응, 비용 최적화 |

## 성능

| 용어 | 기획자용 설명 | 기획 영향 |
|------|-------------|----------|
| **Latency** | 요청부터 응답까지 걸리는 시간 | 사용자 체감 속도 |
| **Throughput** | 단위 시간당 처리 가능한 요청 수 | 동시 사용자 수 |
| **SLA** | 서비스 가용성 보장 수준 (예: 99.9%) | 다운타임 허용 범위, 보상 정책 |
| **Cold Start** | 오랫동안 안 쓰던 기능이 첫 실행 시 느린 현상 | 사용 빈도 낮은 기능의 UX |
