# 팀 간 용어 사전

기획-디자인-개발 팀 간 일관된 용어 사용을 위한 가이드.
이 문서는 프로젝트별로 커스터마이징하여 사용한다.

## 기본 원칙

1. 기획서에는 **기획 용어**를 사용한다
2. 기술 용어는 괄호 안에 병기할 수 있다: "서버 데이터 조회 (GET API)"
3. 동일 개념에는 동일 용어를 사용한다

## 표준 용어 매핑

### UI 상태

| 표준 용어 (기획) | 비표준 (사용 금지) | 기술 용어 (참고) |
|-----------------|-------------------|----------------|
| 로딩 중 | 불러오는 중, 읽는 중 | loading, isLoading |
| 에러 | 오류, 실패 | error, isError |
| 데이터 없음 | 빈 화면, 결과 없음 | empty, no data |
| 정상 표시 | 성공, 완료 | success, loaded |

### 사용자 행동

| 표준 용어 (기획) | 비표준 (사용 금지) | 기술 용어 (참고) |
|-----------------|-------------------|----------------|
| 클릭 | 터치, 탭, 누름 | onClick, click event |
| 입력 | 타이핑, 작성 | onChange, input event |
| 제출 | 전송, 보내기 | onSubmit, form submit |
| 선택 | 고르기, 픽 | onSelect, select |
| 스크롤 | 내리기 | onScroll, scroll event |

### 시스템 동작

| 표준 용어 (기획) | 비표준 (사용 금지) | 기술 용어 (참고) |
|-----------------|-------------------|----------------|
| 화면 이동 | 페이지 전환, 리다이렉트 | navigate, router.push |
| 팝업 표시 | 모달 열기, 다이얼로그 | modal.open, dialog |
| 알림 표시 | 토스트, 스낵바 | toast, notification |
| 서버 요청 | API 호출, 패칭 | fetch, API call |
| 데이터 저장 | 캐싱, 스토어 | cache, store |

### 기능 유형

| 표준 용어 (기획) | 비표준 (사용 금지) | 기술 용어 (참고) |
|-----------------|-------------------|----------------|
| 생성 | 만들기, 추가, 등록 | Create, POST |
| 조회 | 보기, 읽기, 검색 | Read, GET |
| 수정 | 변경, 업데이트, 편집 | Update, PUT/PATCH |
| 삭제 | 제거, 지우기 | Delete, DELETE |
| 목록 조회 | 리스트, 목록 보기 | List, GET (collection) |
| 상세 조회 | 디테일, 상세 보기 | Detail, GET (single) |
| 검색 | 찾기, 서치 | Search, query |
| 필터링 | 거르기, 분류 | Filter, filter params |
| 정렬 | 소팅, 순서 변경 | Sort, order by |
| 페이지네이션 | 페이지 나누기 | Pagination, page/limit |

### 권한/인증

| 표준 용어 (기획) | 비표준 (사용 금지) | 기술 용어 (참고) |
|-----------------|-------------------|----------------|
| 로그인 필요 | 인증 필요, 로그인 상태 | authenticated, auth required |
| 권한 필요 | 접근 권한, 허용 역할 | authorized, role check |
| 관리자 전용 | 어드민 전용 | admin role |
| 권한 없음 | 접근 불가, 미인가 | 403 Forbidden, unauthorized |

## 용어 충돌 해결 규칙

1. **기획서 우선**: 기획서에서 사용하는 용어가 팀 표준
2. **최초 사용자 우선**: 동일 문서 내 먼저 등장한 용어를 일관 사용
3. **합의 기록**: 용어 변경 시 이 문서를 업데이트하고 변경 이력 기록
