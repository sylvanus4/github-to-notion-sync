# Channel Formats

채널별 문구 포맷 가이드. policy-text-generator Phase 2에서 사용.

## Push Notification

```
[제목] (최대 30자)
[본문] (최대 80자)
```

**작성 원칙**:
- 제목: 핵심 행동/상태 변화 1가지만
- 본문: 구체적 정보 + 행동 유도
- 긴급도 표현: "지금", "오늘까지", "곧" 등 시간 표현 활용
- 이모지: 1개까지 허용 (선두 배치)

**예시**:
```
환불 완료 안내
회원님이 요청하신 환불이 처리되었습니다. 3~5영업일 내 입금됩니다.
```

## In-App Message

### 팝업 (Modal)
```
[제목] (최대 40자)
[본문] (최대 200자)
[CTA 버튼 텍스트] (최대 15자)
[보조 링크] (선택)
```

### 배너 (Banner)
```
[한 줄 메시지] (최대 60자) + [CTA] (최대 10자)
```

### 토스트 (Toast)
```
[메시지] (최대 40자)
```

**작성 원칙**:
- 팝업: 중요도 높은 안내, 사용자 동의 필요한 경우
- 배너: 진행 중인 이벤트, 비긴급 안내
- 토스트: 액션 완료 피드백

## Email

```
[제목] (최대 50자, 모바일 프리뷰 고려)
[프리헤더] (최대 100자)

[인사말]
[본문 - 핵심 내용]
[상세 설명 / 표]
[CTA 버튼]
[법적 고지 / 문의처]
[수신거부 링크]
```

**작성 원칙**:
- 제목: 핵심 정보 선두 배치 (모바일 잘림 고려)
- 본문: 3문단 이내, 핵심 먼저
- 법적 고지: 관련 법규 준수 (정보통신망법 등)
- 수신거부: 필수 포함

## Notice (공지/안내문)

```
# [공지 제목]

**시행일**: YYYY-MM-DD
**적용 대상**: [대상 범위]

## 주요 변경 사항
[변경 전/후 비교표]

## 상세 내용
[조항별 설명]

## 문의
[담당부서 / 연락처]

---
본 공지는 [관련 법규/약관 근거] 에 따라 안내드립니다.
```

**작성 원칙**:
- 시행일 필수 명시
- 변경 사항은 Before/After 비교표 사용
- 공식 어조 유지
- 법적 근거 하단 명시

## SMS

```
[발신자] [메시지 본문] (90바이트 = 한글 약 45자)
```

**작성 원칙**:
- 발신번호 표기 필수
- 핵심 정보만 (who, what, when)
- URL은 단축 URL 사용
- 광고성: `(광고)` 표기 + 수신거부 번호 필수
