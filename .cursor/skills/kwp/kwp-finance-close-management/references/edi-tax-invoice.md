# 전자세금계산서 EDI 가이드

> Source: modu-ai/cowork-plugins moai-finance close-management + financial-statements references

## 개요

전자세금계산서는 국세청 홈택스를 통해 발행/수취/수정/취소를 처리하는 디지털 세금계산서입니다.
2024년부터 모든 법인사업자와 직전연도 공급가액 1억원 이상 개인사업자에게 의무화되었습니다.

## 발행 절차

### 1. 홈택스 직접 발행

1. 홈택스 로그인 -> 전자세금계산서 -> 발급
2. 공급자/공급받는자 사업자번호 입력
3. 품목, 수량, 단가, 공급가액, 세액 입력
4. 발급 -> 국세청 전송 (자동)

### 2. ERP/회계 프로그램 연동 (EDI)

1. ERP에서 매출 전표 생성
2. 전자세금계산서 자동 생성 (API 연동)
3. 국세청 ASP 사업자를 통해 전송
4. 전송 결과 확인 (성공/실패)

### 주요 ASP 사업자

| 사업자 | 용도 |
|--------|------|
| 비즈니스온 | 중소기업 |
| 더존 | ERP 연동 |
| 한국정보인증 | 공인인증 기반 |
| 금융결제원 | 금융기관 연동 |

## 수정 세금계산서

수정이 필요한 경우 원본을 취소하고 재발행합니다.

| 사유 | 수정 유형 | 기한 |
|------|----------|------|
| 기재사항 착오 | 기재사항 착오 정정 | 확정신고 기한 |
| 공급가액 변동 | 공급가액 변동 | 변동 사유 발생일 |
| 환입 (반품) | 환입 | 환입일 |
| 계약 해제 | 계약의 해제 | 해제일 |
| 착오에 의한 이중 발급 | 착오에 의한 이중 발급 | 확정신고 기한 |

## 가산세 기준

| 위반 유형 | 가산세 |
|----------|--------|
| 미발급 | 공급가액의 2% |
| 지연 발급 (1개월 이내) | 공급가액의 1% |
| 전자 외 발급 (종이) | 공급가액의 1% |
| 미전송 (국세청) | 공급가액의 0.5% |

## 월말 결산 체크포인트

전자세금계산서 관련 월말 결산 시 확인사항:

### T+1 (마감일 다음날)

- [ ] 당월 매출 전자세금계산서 전건 발행 완료 확인
- [ ] 미전송 건 홈택스에서 일괄 확인 (전송 현황 조회)
- [ ] 매입 세금계산서 수취 현황 대사

### T+3 (마감일 +3일)

- [ ] 매출/매입 세금계산서 건수 및 금액 ERP 대사
- [ ] 수정 세금계산서 발행 필요 건 식별 및 처리
- [ ] 공급받는자 사업자번호 휴폐업 조회 (국세청 사업자등록상태조회)

### 익월 10일 전 (가산세 방지)

- [ ] 당월 매출분 세금계산서 전건 발행 완료 (지연 가산세 1% 방지)
- [ ] 미전송 건 재전송 처리
- [ ] 부가세 예정/확정 신고 대비 매출/매입 최종 대사

## 부가세 신고 연계

| 항목 | 내용 |
|------|------|
| 신고 주기 | 법인: 분기, 개인: 반기 (간이과세자: 연 1회) |
| 매출 세금계산서 | 부가세 매출세액 산출 기초 |
| 매입 세금계산서 | 매입세액 공제 근거 |
| 전자발행 세액공제 | 개인사업자 건당 200원 (연간 100만원 한도) |

## 실무 체크리스트

- [ ] 매출 발생 시 익월 10일까지 발행 (지연 가산세 방지)
- [ ] 공급받는자 사업자번호 사전 확인 (휴폐업 조회)
- [ ] 수정 세금계산서는 원본 번호 참조하여 발행
- [ ] 월말 미전송 건 일괄 확인 (홈택스 -> 전송 현황 조회)
- [ ] 부가세 신고 전 매출/매입 세금계산서 대사 완료
- [ ] ERP 연동 시 ASP 전송 실패 건 자동 알림 설정
- [ ] 수정 세금계산서 발행 시 원본 세금계산서 승인번호 기록
