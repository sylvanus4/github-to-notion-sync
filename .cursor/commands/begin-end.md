# Bookend Response Format

Force every response to start with a specific opening phrase and end with a specific closing phrase. Useful for consistent formatting, branding, or structured output.

## Usage

```
/begin-end begin:TL;DR: end:— Generated by AI Assistant What are the key risks of serverless?
/begin-end begin:⚠️ DRAFT — NOT REVIEWED end:--- End of draft --- Write a security policy for API keys
/begin-end begin:## Executive Summary end:## Next Steps Design our Q4 product roadmap
/begin-end begin:환자 여러분께 알립니다. end:감사합니다. 의료팀 드림 진료 시간 변경 안내문 작성
```

## Your Task

User input: $ARGUMENTS

### Workflow

1. **Parse tokens:**
   - Extract text after `begin:` until the next `end:` token → opening phrase
   - Extract text after `end:` until the actual question begins → closing phrase
   - Everything remaining → the actual question to answer
2. **Generate response body** — Answer the question fully
3. **Prepend** the opening phrase exactly as specified
4. **Append** the closing phrase exactly as specified
5. **Verify** — Confirm the first line starts with the opening and the last line contains the closing

### Output Format

```
[Opening phrase — exactly as specified]

[Response body]

[Closing phrase — exactly as specified]
```

### Constraints

- Bookend phrases must be preserved verbatim — no paraphrasing
- The response body must still be high quality — bookends don't reduce content standards
- If `begin:` or `end:` tokens are missing, prompt the user for the correct format
- Nested bookends are not supported — only one pair per invocation
