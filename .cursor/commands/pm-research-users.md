# PM Research Users

Run a comprehensive user research pipeline: personas, segmentation, journey maps from interview notes, surveys, or behavioral data. Uses pm-market-research skill with user-personas, user-segmentation, market-segments, customer-journey-map sub-skills.

## Usage
```
/pm-research-users Create personas from these interview notes
/pm-research-users 인터뷰 노트에서 페르소나 만들어줘
/pm-research-users Segment users by behavior and demographics
/pm-research-users 사용자 세분화 및 여정 맵
/pm-research-users Journey map for new customer onboarding
/pm-research-users 신규 고객 온보딩 여정 맵
```

## Workflow

### Step 1: Ingest Research Data
- Accept: interview notes, survey responses, support tickets, analytics summaries
- If no data: request sample or work from stated assumptions (tag as "assumption-based")

### Step 2: User Personas
- Identify 3–5 distinct personas
- For each: name, demographics, goals, pain points, behaviors, quotes
- Prioritize by addressable market or strategic importance

### Step 3: Segmentation
- Define segmentation criteria: behavioral (usage, frequency), demographic (role, company size), psychographic (goals, values)
- Create segments with clear boundaries
- Note segment size and growth potential if inferrable

### Step 4: Customer Journey Map
- Phases: Awareness → Consideration → Purchase → Onboarding → Retention → Advocacy
- For each phase: actions, touchpoints, emotions, pain points, opportunities
- Map to personas or segments where relevant

### Step 5: Synthesis
- Highlight top 3 insights and recommended actions
- Link personas and segments to product/feature implications
- Flag gaps in research (missing segments, unvalidated assumptions)

## Notes
- Personas without research backing should be labeled "hypothesis" — validate before investing
- Journey maps work best when tied to a specific persona or segment
- Keep personas to 3–5; more dilutes focus
