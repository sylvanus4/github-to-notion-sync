# PM Competitive Analysis

Analyze the competitive landscape: identify competitors, compare strengths and weaknesses, and surface differentiation opportunities. Include a feature comparison matrix and positioning map. Uses pm-market-research skill, competitor-analysis sub-skill.

## Usage
```
/pm-competitive-analysis Analyze competitors for our B2B analytics product
/pm-competitive-analysis 우리 B2B 분석 제품 경쟁사 분석
/pm-competitive-analysis Who competes in the AI coding assistant space?
/pm-competitive-analysis AI 코딩 어시스턴트 시장 경쟁사
/pm-competitive-analysis Feature matrix vs Datadog, Grafana, New Relic
/pm-competitive-analysis 기능 비교 매트릭스 만들어줘
```

## Workflow

### Step 1: Define Scope
- Product or market segment to analyze
- Geographic scope (global, regional)
- Time horizon (current state, 12–18 month outlook)

### Step 2: Identify Competitors
- **Direct**: Same product category, same buyer
- **Indirect**: Different category but solves same job (e.g., spreadsheet vs BI tool)
- **Emerging**: New entrants, adjacent players expanding
- Include 5–10 competitors for depth

### Step 3: Strengths and Weaknesses
- For each competitor: key strengths (why customers choose), key weaknesses (switching triggers)
- Use evidence: reviews, analyst reports, pricing, feature sets
- Note positioning and messaging themes

### Step 4: Feature Comparison Matrix
- Rows: features/capabilities (grouped by theme)
- Columns: our product + each competitor
- Values: Full / Partial / None or score (1–5)
- Highlight differentiators and gaps

### Step 5: Positioning Map
- 2×2 or 3×3 map on meaningful axes (e.g., Ease of use vs Power, Price vs Features)
- Plot our product and competitors
- Identify white space (underserved quadrants)

### Step 6: Differentiation Opportunities
- Where can we win? (unique capability, underserved segment, better UX)
- What must we match? (table stakes)
- What to avoid? (crowded, commodity areas)

## Notes
- Use public sources; avoid confidential competitor intel unless explicitly provided
- Feature matrix: focus on decision-relevant features, not exhaustive lists
- Positioning map axes should resonate with buyer priorities
